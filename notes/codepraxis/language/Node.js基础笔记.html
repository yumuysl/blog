一、precess模块

以下是 Node.js process 核心模块的常用方法及案例说明：

一、基础信息获取

  1. process.argv
    获取命令行参数（返回数组）

    // node app.js --name=Aliyun --port=3000
console.log(process.argv); 
// 输出: [node路径, 脚本路径, '--name=Aliyun', '--port=3000']

  2. process.env
    访问环境变量

    // 读取环境变量（常用于区分开发/生产环境）
console.log(process.env.NODE_ENV); 
// 输出 development 或 production

  3. process.cwd()
    获取当前工作目录

    console.log(process.cwd()); 
// 输出如: /Users/yourname/projects

二、进程控制

  1. process.exit([code])
    强制退出进程（默认 code=0）

    if (error) {
  process.exit(1); // 非0表示异常退出
}

  2. process.exitCode
    设置退出码（更安全的替代方案）

    process.exitCode = 1; // 进程结束后自动以 code=1 退出

  3. process.kill(pid[, signal])
    发送信号给进程

    process.kill(process.pid, 'SIGTERM'); // 终止当前进程

三、事件监听

  1. 'exit' 事件
    进程退出时触发（同步操作）

    process.on('exit', (code) => {
  console.log(`进程退出，code: ${code}`);
});

  2. 'uncaughtException' 事件
    捕获未处理的异常

    process.on('uncaughtException', (err) => {
  console.error('捕获到未处理异常:', err);
  process.exit(1);
});

  3. 'SIGINT' 信号
    监听 Ctrl+C 操作

    process.on('SIGINT', () => {
  console.log('收到终止信号，优雅退出');
  server.close();
});

四、实用工具

  1. process.nextTick(callback)
    将回调放入事件循环队列头部

    process.nextTick(() => {
  console.log('这个回调会优先执行');
});

  2. process.memoryUsage()
    获取内存使用情况

    const { rss, heapTotal } = process.memoryUsage();
console.log(`内存占用: ${rss} bytes`);

  3. process.platform
    检测操作系统类型

    if (process.platform === 'win32') {
  console.log('Windows 系统');
}

粘包问题以及解决方案

  1. 粘包问题：由于数据传输速率以及读取写入均有差异，接收端与发送端设有缓冲区，这些缓冲区用于临时存放待读取/写入的数据，提高主机/服务器性能，这样同时也导致一个问题，当数据到缓冲区后数据并不会被立刻消耗而是等待缓冲区数据足够多后一次性消耗，这样数据会粘在一起，这就是粘包问题。
  2. 解决方法：广泛使用的一个方法是给消息头部设置长度，然后读取相应长度作为区分，具体封包解包代码如下:

封装

class MyTransfromCode{
  constructor(){
    this.packageHeaderLen = 4;
    this.seriaNum = 0;
    this.serialLen = 2;
  }

  /**
  * 编码
  * @Description:
  *  1. 先按照指定的长度来申请一片内存空间作为header
  *  2. 通过writeInt16BE方法整体写入内容
  *  3. 连接header和数据内容
    **/
  encode(data,serialNum){
    const body = Buffer.from(data);
    const headerBuf = Buffer.alloc(this.packageHeaderLen);
    headerBuf.writeInt16BE(serialNum || this.seriaNum);
    headerBuf.writeInt16BE(body.length, this.serialLen);
    if(serialNum == undefined){
      this.serialNum++;
    }

    return Buffer.concat([headerBuf, body])
  }

  /**
  * 解码
  * @Description:
  *  1. 截取header长度
  *  2. 截取内容长度
  *  3. 通过readInt16BE()方法解码，返回数据
  **/
  decode(buffer){
    const headerBuf = buffer.subarray(0, this.packageHeaderLen);
    const bodyBuf = buffer.subarray(this.packageHeaderLen);

    return {
      seriaNum: headerBuf.readInt16BE(),
      bodyLength: headerBuf.readInt16BE(this.serialLen),
      body: bodyBuf.toString()
    }
  }

  /**
  * 获取包大小
  * @Description:
  *  1. 当buffer数据长度小于消息头长度时，说明还未传输完毕
  **/
  getPackageLen(buffer){
    if(buffer.length < this.packageHeaderLen){
      return 0;
    }else{
      return this.packageHeaderLen + buffer.readInt16BE(this.serialLen)
    }
  }
}

module.exports = MyTransfromCode;

十一、Cluster 模块

Node.js 的 cluster 模块允许你轻松创建共享同一服务器端口的子进程（worker processes）。由于 Node.js 是单线程的，cluster 模块可以帮助我们充分利用多核 CPU 系统的性能。 简单来说，cluster 模块就像是一家餐厅的经理，它创建多个服务员（worker）来处理顾客（请求），而不是让一个服务员处理所有顾客

为什么需要Cluster模块

  • 提高性能：现代 CPU 通常有多个核心，但 Node.js 默认只使用一个核心。cluster 模块可以让你利用所有可用的 CPU 核心。
  • 提高可靠性：如果一个 worker 崩溃了，其他 worker 可以继续处理请求，主进程可以重新启动崩溃的 worker。
  • 零停机时间：可以逐个重启 worker 来实现代码更新，而不会中断服务。

Cluster 模块基础用法

const cluster = require('cluster');
const http = require('http');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  console.log(`主进程 ${process.pid} 正在运行`);

  // 衍生 worker 进程
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }

  cluster.on('exit', (worker, code, signal) => {
    console.log(`worker ${worker.process.pid} 已退出`);
  });
} else {
  // Worker 可以共享同一个 TCP 连接
  // 这里是一个 HTTP 服务器
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('你好世界\n');
  }).listen(8000);

  console.log(`Worker ${process.pid} 已启动`);
}

常用API

  • cluster.fork()：创建一个新的 worker 进程。调用这个方法会返回一个 worker 对象
  • cluster.isMaster：如果是主进程，则为 true
  • cluster.isWorker：如果是 worker 进程，则为 true
  • cluster.workers：包含所有活跃 worker 对象的哈希表，以 worker.id 作为键名

应用场景

  • 负载均衡
// 在主进程中
const cluster = require('cluster');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
} else {
  require('./app'); // 你的应用入口文件
}
  • 优雅重启
cluster.on('exit', (worker, code, signal) => {
  console.log(`Worker ${worker.process.pid} 退出`);
  // 可以在这里重新 fork 一个新的 worker
  cluster.fork();
});
  • 进程间通信
  // 主进程发送消息
worker.send('Hello worker!');

// Worker 接收消息
process.on('message', (msg) => {
  console.log(`Worker ${process.pid} 收到消息: ${msg}`);
});

Cluster 模块的替代方案

虽然 cluster 模块很强大，但也有其他选择：

  • PM2：进程管理器，内置负载均衡和监控功能
  • Nginx：反向代理，可以在多个 Node.js 实例间负载均衡
  • Kubernete：容器编排系统，可以自动扩展 Node.js 服务
