组件和hook

  • react中的组件其实就是一个function，retrun返回数据再导出供其他文件调用此组件
  • react hooks使用规则：不要在循环、条件、嵌套函数或 try/catch/finally 块中调用 Hook。相反，在任何早期返回之前，总是在 React 函数的顶层使用 Hooks。“顶层调用”​ 指的是 Hook 调用语句本身必须在函数体的顶层作用域，且每次渲染时都以相同的顺序被调用，可以在调用Hook之前执行普通js代码
  • useRef：获取dom元素
  • useEffect：组件渲染后执行，类似生命周期，一般用来处理异步操作
  • useMemo：缓存计算结果，避免重复计算
  • useCallback：缓存函数，避免重复创建函数
  • useContext：跨组件通信

受控组件和非受控组件

受控组件

  • 由react的state控制的表单组件，适合频繁更新、复杂数据校验的场景，如：
import React, { useState } from 'react';

const ControlledForm = () => {
  const [value, setValue] = useState('');

  const handleChange = (e) => {
    setValue(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log('Form submitted with value:', value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input type="text" value={value} onChange={handleChange} />
      <button type="submit">Submit</button>
    </form>
  );
};

export default ControlledForm;

useEffect()

  1. 可以用来监听组件状态的变化，并在发生变化时执行一些代码。
  2. useEffect 的第一个参数是一个函数，该函数会在组件挂载时执行一次，并且在组件状态发生变化时再次执行。这个函数可以用来获取组件状态，并在状态发生变化时执行一些代码。
  3. useEffect 的第二个参数是一个依赖项数组，可以为空，为空表示在组件挂在时执行一次。不为空，那么 useEffect 函数只会在这些状态发生变化时执行
  4. 清理函数：在useEffect的第一个参数函数中，其返回值可选，可以返回一个回调函数，在组件卸载时执行，用于清理相关变量，防止内存泄漏，比如清理计时器
import React, { useState, useEffect } from 'react';

const DataFetcher = () => {
  const [data, setData] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      const response = await fetch('/api/data');
      const json = await response.json();
      setData(json);
    };

    fetchData();

    // 清理函数
    return () => {
      console.log('Cleanup');
    };
  }, []); // 依赖数组为空，表示仅在组件挂载时执行

  return (
    <div>
      {data ? <pre>{JSON.stringify(data, null, 2)}</pre> : <p>Loading...</p>}
    </div>
  );
};

export default DataFetcher;

useMemo

  • 返回一个缓存的值，只有当依赖项发生变化时，才会重新计算这个值，避免不必要的重复计算或渲染
  • 避免过早优化。只有在计算确实非常昂贵，且依赖项不频繁变化时
import React, { useMemo, useState } from 'react';

function ExpensiveCalculationComponent({ list }) {
  const [filter, setFilter] = useState('');
  const [count, setCount] = useState(0);

  // 使用 useMemo 记忆计算结果，只有当 list 或 filter 变化时，才会重新计算 filteredList
  const filteredList = useMemo(() => {
    console.log('正在进行昂贵的计算...');
    return list.filter(item => item.includes(filter));
  }, [list, filter]); // 依赖项：list, filter

  return (
    <div>
      <input value={filter} onChange={(e) => setFilter(e.target.value)} />
      <div>{filteredList.map(item => <p key={item}>{item}</p>)}</div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Re-render</button>
    </div>
  );
}

useCallback

  • 返回一个已经缓存的回调函数，当依赖数组方式变化时，函数才会重新创建，避免不必要的重复计算或渲染
import React, { useCallback, useState } from 'react';
import ExpensiveChildComponent from './ExpensiveChildComponent';

function ParentComponent() {
  const [count, setCount] = useState(0);
  const [text, setText] = useState('');

  // 使用 useCallback 记忆函数，只有当 count 变化时，onButtonClick 才会改变
  const onButtonClick = useCallback(() => {
    setCount(c => c + 1);
  }, [count]); // 依赖项：count

  return (
    <div>
      <input value={text} onChange={(e) => setText(e.target.value)} />
      <ExpensiveChildComponent onButtonClick={onButtonClick} />
    </div>
  );
}

useLayoutEffect

  • useLayoutEffect的函数签名与 useEffect完全相同，但执行时机不同。它会在所有的 DOM 变更之后、浏览器绘制页面之前同步执行
  • 用法：useLayoutEffect(() => { ... }, [dependencies]);
  • 适用场景：当需要直接读取或操作 DOM（例如测量元素的尺寸、样式），并且希望在下一次浏览器绘制之前完成这些操作，以避免页面闪烁时使用。
  • 注意事项：由于是同步执行，内部的更新会阻塞浏览器绘制。应尽量避免在此 Hook 内执行耗时操作或复杂逻辑
function Tooltip() {
  const ref = useRef(null);
  const [width, setWidth] = useState(0);

  useLayoutEffect(() => {
    // 在浏览器绘制前测量 DOM 元素的宽度
    if (ref.current) {
      const measuredWidth = ref.current.getBoundingClientRect().width;
      setWidth(measuredWidth);
    }
  }, []); // 依赖项为空，仅在挂载后执行一次

  return <div ref={ref}>这个工具的宽度是：{width}px</div>;
}

自定义hook

  • 自定义 Hook 的核心思想是将组件中可复用的状态逻辑提取到一个独立的函数中，从而实现逻辑的复用和解耦
  • 导出导入自定义hook
  • 和普通函数区别
    • 必须以use开头，而普通函数无要求
    • ​可以调用其他 React Hook，而普通函数不可以
    • 可管理有状态的逻辑和副作用（如组件生命周期、事件监听），而普通函数不可以
    • 在不同组件中调用，状态完全独立，而普通函数内部变量通常在所有调用间共享（模块级作用域）
    • 复用有状态的组件逻辑，是组件逻辑的延伸，而普通函数复用无状态的工具逻辑，实现代码复用

重定向
  • 重定向是指将用户从一个路由自动导航到另一个路由。
  • 要实现重定向，我们可以使用 Redirect 组件。 Redirect 组件接受一个 to 属性，该属性指定要重定向到的路径。
  • 例如，我们可以创建一个重定向路由，将用户从 /login 自动导航到 :
import React from "react";
import { Route, Redirect } from "react-router";

const Login = () => {
  // Simulate user authentication
  const isAuthenticated = true;

  if (isAuthenticated) {
    return <Redirect to="/dashboard" />;
  }

  return <h1>Login</h1>;
};

const Dashboard = () => {
  return <h1>Dashboard</h1>;
};

const App = () => {
  return (
    <div>
      <Route path="/login" component={Login} />
      <Route path="/dashboard" component={Dashboard} />
    </div>
  );
};

export default App;

状态管理进阶

  • 前面只记录了使用useState钩子管理状态，这种方式称为组件状态，但除了组件状态外，还有状态容器和第三方状态管理库两种方法管理状态。
状态容器
  • 状态容器是指一个单独的组件，它负责管理应用程序的所有状态。状态容器可以使用 useContext 钩子来访问应用程序的所有状态。
import React, { useState, useContext } from "react";

const AppContext = React.createContext({});

const App = () => {

  return (
    <AppContext.Provider value={count}>
      <Counter />
    </AppContext.Provider>
  );
};

const Counter = () => {
  const count = useContext(AppContext);

  return (

  );
};

export default App;

异步状态管理：TanStack Query

为什么需要异步状态管理

  • 在useEffect中调用API获取数据是相对常见的模式，但会存在一些问题：
    • 性能：数据获取在渲染完成后才开始，导致不必要的等待
    • 数据一致性：快速切换依赖项（如列表筛选条件）时，先发起的请求可能后返回，导致显示数据与当前状态不匹配
    • 状态管理复杂：需手动管理加载中 (loading)、错误 (error)、数据 (data) 等多种状态，代码冗长且易出错
    • 数据缓存：无法自动缓存数据，每次组件挂载都会重新请求，缺乏后台自动重新验证等高级功能
  • 针对上述问题常见，使用专为数据获取的库更为稳妥，如TanStack Query，优点如下：
    • 简化代码：用一个 Hook封装了数据、加载状态、错误信息等
    • 内置智能缓存：相同数据的请求会自动被缓存和复用，无需重复发起
    • 自动处理竞态和重复请求：库内部已经处理了棘手的问题

import {
  useQuery,
  useMutation,
  useQueryClient,
  QueryClient,
  QueryClientProvider,
} from '@tanstack/react-query'
import { getTodos, postTodo } from '../my-api'

// Create a client
const queryClient = new QueryClient()

function App() {
  return (
    // Provide the client to your App
    <QueryClientProvider client={queryClient}>
      <Todos />
    </QueryClientProvider>
  )
}

function Todos() {
  // Access the client
  const queryClient = useQueryClient()

  // Queries
  const query = useQuery({ queryKey: ['todos'], queryFn: getTodos })

  // Mutations
  const mutation = useMutation({
    mutationFn: postTodo,
    onSuccess: () => {
      // Invalidate and refetch
      queryClient.invalidateQueries({ queryKey: ['todos'] })
    },
  })

  return (
    <div>
      <ul>
        {query.data?.map((todo) => (
          <li key={todo.id}>{todo.title}</li>
        ))}
      </ul>

      <button
        onClick={() => {
          mutation.mutate({
            id: Date.now(),
            title: 'Do Laundry',
          })
        }}
      >
        Add Todo
      </button>
    </div>
  )
}

render(<App />, document.getElementById('root'))

React面试题

核心概念与基础题

React的核心特性是什么？虚拟DOM如何提升性能？

  • 核心特性：包括组件化、声明式UI、虚拟DOM和单向数据流。声明式编程让开发者只需描述UI应呈现的状态，而不必一步步操作DOM
  • 虚拟dom提升性能：在内存中存一个轻量级的dom树（就是一个js对象）
    • 创建新的虚拟DOM树
    • Diff算法比对：当状态改变时，react会对比新的dom和旧的dom，找出最小差异
    • 批量更新与最小化操作：将计算出的差异批量、精准地应用到真实DOM上，减少直接操作DOM带来的性能损耗

什么是JSX？它和HTML有什么主要区别？

  • JSX：是JavaScript的语法扩展，允许在JS中编写类似HTML的结构。它最终会被编译（如通过Babel）为 React.createElement()调用，生成虚拟DOM对象
  • 与HTML区别
    • 属性命名：jsx是驼峰命名，而html是全小写
    • 动态内容嵌入：使用大括号{}嵌入js表达式
    • 样式：jsx中是使用一个样式对象而非css

受控组件与非受控组件的区别是什么？分别适用于什么场景？

  • 受控组件：表单项的值由React的state控制， onChange 等事件处理器用于更新state。数据流是双向的（state变化驱动UI更新，UI交互更新state）。适用于需要实时验证、即时提交或根据输入动态控制其他UI元素的场景
  • 非受控组件：表单项的值由DOM自身管理，通常通过 ref来获取DOM值。更像是传统的HTML表单。适用于表单简单、无需实时验证、或需要集成非React代码的场景，如文件上传

React中state和props有何不同？

  • props：是父组件传递给子组件的只读数据。子组件不能直接修改接收到的props。用于组件间通信和数据传递
  • state：是组件内部管理的、可变的私有数据。通过 setState（类组件）或 useStateHook的setter函数（函数组件）更新。state变化会触发组件重新渲染。用于管理组件自身的状态，如用户输入、UI状态

Hooks与函数组件题

解释useState和useEffect的用途。useEffect的依赖数组如何工作？

  • useState：用于在函数组件中添加内部state。返回一个状态变量和一个用于更新该状态的函数
  • useEffect：用于在函数组件中执行副作用操作，如数据获取、订阅、手动修改DOM等。它接收一个副作用函数和一个可选的依赖数组，其中副作用函数可以返回一个清理函数，防止内存泄露
    • 不提供数组：副作用函数在每次渲染后都执行
    • 空数组 []：副作用函数仅在组件挂载后执行一次，类似于 componentDidMount
    • 包含特定依赖的数组 [dep1, dep2]：只有当依赖项的值发生变化时，副作用函数才会执行

useCallback和useMemo有什么区别？分别在什么场景下使用？

  • useMemo：用于缓存昂贵的计算结果。只有当其依赖项改变时，才会重新计算记忆化的值。避免在每次渲染时都进行复杂计算
  • useCallback：用于缓存函数本身。只有当其依赖项改变时，才会返回一个新的函数引用。常用于避免因函数引用变化导致子组件不必要的重新渲染。

在React中如何共享逻辑？自定义Hook是什么？

  • 逻辑复用方式：高阶组件、Render Props、自定义Hook
  • 自定义Hook：是一个以 use开头的JavaScript函数，它内部可以调用其他Hook。用于将组件逻辑提取到可重用的函数中，解决逻辑共享问题

为什么不能在循环、条件或嵌套函数中调用Hook？

  • 根本原因：React 依赖于Hook的调用顺序来正确管理Hook的状态。如果在循环、条件或嵌套函数中调用Hook，会破坏Hook的调用顺序，导致状态对应错误，引发bug

组件设计与生命周期

React组件的生命周期有哪些主要阶段？在函数组件中如何模拟生命周期方法？

  • 主要阶段：挂载（Mounting）、更新（Updating）、卸载（Unmounting）。
  • 函数组件中的模拟：
    • componentDidMount：useEffect(fn, [])。
    • componentDidUpdate：useEffect(fn)或 useEffect(fn, [deps])。
    • componentWillUnmount：useEffect的清理函数 return () => { ... }

什么是高阶组件（HOC）或Render Props？与Hook相比有何优劣？

什么是错误边界（Error Boundary）？它不能捕获哪些错误？

  • 错误边界：是React组件，用于捕获其子组件树中JavaScript错误，并记录这些错误、显示降级UI，而不是崩溃整个组件树。通过定义 static getDerivedStateFromError()或 componentDidCatch()生命周期方法的类组件实现。

  • 不能捕获的错误：事件处理函数内部的错误、异步代码（如setTimeout）、服务端渲染（SSR）中的错误、错误边界组件自身抛出的错误

性能优化题

有哪些方法可以避免React组件不必要的重新渲染？

  • React.memo：用于函数组件，对props进行浅比较，避免相同props下的不必要重渲染。
  • PureComponent：用于类组件，默认实现了shouldComponentUpdate中对props和state的浅比较。
  • useMemo / useCallback：缓存计算值和函数，避免因引用变化导致子组件重渲染

渲染列表时，为什么需要key？为什么不宜用数组索引作为key？

  • key的作用：key帮助React识别哪些列表项发生了变化（被添加、删除或重排），以便高效地更新和复用DOM元素
  • 避免使用索引作为key的原因：当列表发生插入、删除或重排序时，数组索引会发生变化，这可能导致React错误地复用DOM元素（造成状态错乱）、性能下降（不必要的重新渲染）

React中的setState是同步还是异步的？在什么情况下会是同步的？

  • 在React事件处理函数中，setState通常是异步的。React会对多个setState调用进行批处理，然后统一更新，以避免不必要的重复渲染。
  • 在React 18及以后，在事件处理函数、useEffect等预期内的更新源中，React默认会自动进行批处理。
  • 在setTimeout、原生事件监听器等非React上下文环境中，setState可能是同步的
