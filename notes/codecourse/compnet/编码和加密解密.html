数据传输过程中编码转换

Node.js 数据传输编码转换指南

一、基础编码转换

  1. 二进制数据和编码格式关系

    • 二进制数据：计算机内部表示数据的方式，通常以字节（8位）为单位。
    • 编码格式：将二进制数据转换为人类可读的文本形式。常见的编码格式有 UTF-8、ASCII、Latin1、Base64、Hex（十六进制）等。
    • 二进制文件存储原始字节，无需字符编码，当通过编辑器打开文件时字节流会被强制按字符编码（如UTF-8）解释。

  2. UTF-8 ↔ 其他编码（ASCII、Latin1）

    const buffer = Buffer.from("你好, Node.js!", 'utf-8');
// 转成 Latin1（若包含非拉丁字符会丢失数据）
const latin1Str = buffer.toString('latin1'); 
// 转回 UTF-8
const decodedBuffer = Buffer.from(latin1Str, 'latin1');
console.log(decodedBuffer.toString('utf-8')); // 可能乱码

  3. Base64 编码/解码

    // 编码
const base64 = Buffer.from('Hello Base64').toString('base64'); // SGVsbG8gQmFzZTY0
// 解码
const original = Buffer.from(base64, 'base64').toString(); // Hello Base64

  4. Hex（十六进制）转换

    const hex = Buffer.from('Hello Hex').toString('hex'); // 48656c6c6f20486578
const decoded = Buffer.from(hex, 'hex').toString(); // Hello Hex

二、处理中文编码（GBK/GB2312）

需使用第三方库 iconv-lite（推荐）：

npm install iconv-lite
const iconv = require('iconv-lite');

// UTF-8 → GBK
const gbkBuffer = iconv.encode('你好GBK', 'gbk');

// GBK → UTF-8
const utf8Str = iconv.decode(gbkBuffer, 'gbk');
console.log(utf8Str); // 你好GBK

// 处理 HTTP 请求中的 GBK 响应示例
const axios = require('axios');
const response = await axios.get('http://example.com/gbk-page', { responseType: 'arraybuffer' });
const decodedContent = iconv.decode(Buffer.from(response.data), 'gbk');

三、二进制 ↔ 文本转换

  1. 文本转二进制

    const binaryData = Buffer.from('Hello Binary', 'utf-8');

  2. 二进制转文本

    const text = binaryData.toString('utf-8');

  3. 处理图片等二进制文件

    const fs = require('fs');
// 图片转 Base64
const imageBuffer = fs.readFileSync('image.jpg');
const base64Image = imageBuffer.toString('base64');
// Base64 转回二进制
const decodedImage = Buffer.from(base64Image, 'base64');
fs.writeFileSync('decoded-image.jpg', decodedImage);

四、流处理中的编码转换

使用 stream.Transform 处理大文件：

const { Transform } = require('stream');
const iconv = require('iconv-lite');

// 创建转换流：UTF-8 → GBK
const gbkTransformStream = new Transform({
  transform(chunk, encoding, callback) {
    const gbkChunk = iconv.encode(chunk.toString(), 'gbk');
    this.push(gbkChunk);
    callback();
  }
});

// 使用示例
const fs = require('fs');
fs.createReadStream('input-utf8.txt')
  .pipe(gbkTransformStream)
  .pipe(fs.createWriteStream('output-gbk.txt'));

五、其他编码工具

  1. URL 编码

    const encodedURL = encodeURIComponent('https://example.com/搜索?q=Node.js');
const decodedURL = decodeURIComponent(encodedURL);

  2. 处理日文编码（Shift_JIS）

    npm install encoding-japanese
    const Encoding = require('encoding-japanese');
const sjisBuffer = Encoding.convert(
  Buffer.from('こんにちは'), 
  { to: 'SJIS', from: 'UTF8' }
);

注意事项

  1. 编码一致性：确保发送方和接收方使用相同的字符集
  2. 第三方库选择：优先使用维护活跃的库（如 iconv-lite）
  3. 内存管理：处理大文件时使用流式处理

加密解密(Node.js版本)

一、加密解密基础分类

  1. 对称加密

    • 使用相同密钥加密解密
    • 常见算法：AES（安全）、DES（不安全）、3DES（不安全）
    const crypto = require('crypto');
// AES-256-CBC加密
function encrypt(text, key, iv) {
  const cipher = crypto.createCipheriv('aes-256-cbc', key, iv);
  let encrypted = cipher.update(text, 'utf8', 'hex');
  encrypted += cipher.final('hex');
  return encrypted;
}
// AES-256-CBC解密
function decrypt(encryptedText, key, iv) {
  const decipher = crypto.createDecipheriv('aes-256-cbc', key, iv);
  let decrypted = decipher.update(encryptedText, 'hex', 'utf8');
  decrypted += decipher.final('utf8');
  return decrypted;
}
    • AES加密细则：
      • 初始向量（IV）：随机生成，每次加密都不同
      • 密钥长度：128位、192位、256位
      • 加密模式：
        • CBC：每个明文块与前一个密文块进行异或操作后再加密，需要初始化向量（IV），适合大文件
        • ECB:每个明文块独立加密，不需要iv，不推荐使用
        • CFB：使用前一个密文块作为下一个块的输入，适合流式传输，需要iv
        • OFB：使用前一个密文块作为下一个块的输入，适合流式传输，需要iv
        • CTR：使用计数器生成密钥流，与明文进行异或操作，不需要IV，适合并行处理
        • GCM：支持认证加密（同时提供加密和完整性验证），需要IV，适合需要高安全性的场景
      • 填充模式：
        • 什么是填充模式：AES加密算法要求数据长度必须是块大小的整数倍（例如AES的块大小为128位），当数据长度不足时，需要通过填充模式来补齐。
        • PKCS7：最常用的填充模式，填充字节的值等于需要填充的字节数，通用场景，推荐使用
        • PKCS5：与PKCS7相同，填充字节的值等于需要填充的字节数
        • ZeroPadding：用0x00填充，直到数据长度达到块大小的整数倍，适用于某些特定场景，但可能存在歧义（无法区分填充和数据）。
        • NoPadding：进行填充，要求数据长度必须是块大小的整数倍，适合数据长度已知且为块大小的整数倍的场景

  2. 非对称加密

    • 公钥加密，私钥解密（或反之）
    • 常见算法：RSA、ECC
    const crypto = require('crypto');
// 生成RSA密钥对
const { publicKey, privateKey } = crypto.generateKeyPairSync('rsa', {
  modulusLength: 2048,
});
// 公钥加密
const encryptedData = crypto.publicEncrypt(publicKey, Buffer.from('Hello RSA')).toString('base64');
// 私钥解密
const decryptedData = crypto.privateDecrypt(privateKey, Buffer.from(encryptedData, 'base64')).toString();
    • RES加密细则：
      • 安全性：基于大整数分解的数学难题，安全性较高
      • 适用场景：适合加密小数据（如密钥、短消息），因为 RSA 加密的数据长度受密钥长度限制。
      • 密钥长度：1024位、2048位、3072位
      • 加密模式：
        • PKCS1 v1.5：最常用的填充模式，兼容性好，通用场景，适合加密小数据（如密钥、短消息）。
        • OAEP：更安全的填充模式，推荐使用，安全性要求较高的场景。
        • No Padding：不进行填充，要求数据长度必须是密钥长度的整数倍，适合数据长度已知且为密钥长度的整数倍的场景。
      • 代码示例（采用 OAEP 填充）：
      const encrypted = crypto.publicEncrypt(
  {
    key: publicKey,
    padding: crypto.constants.RSA_PKCS1_OAEP_PADDING,
    oaepHash: 'sha256', // 指定哈希算法
  },
  Buffer.from('Hello RSA')
);
    • ECC加密细则：
      • 特点：
      • 基于椭圆曲线的加密算法，速度快，安全性高
      • 加密、解密和签名速度更快，适合移动设备和物联网设备
      • 密钥长度短，例如256 位 ECC 密钥的安全性相当于 3072 位 RSA 密钥。
      • 适合加密小数据（如密钥、短消息），以及数字签名和密钥交换。
      • 曲线选择
      • secp256k1
      • prime256v1
      • 密钥管理：公钥可以公开，私钥必须严格保密。
      • 性能优化：ECC 加密和解密速度较快，适合资源受限的环境。
      • 常见应用场景：
      • 数字签名：如比特币和以太坊的交易签名。
      • 密钥交换：如 TLS/SSL 中的 ECDHE（Elliptic Curve Diffie-Hellman Ephemeral）。
      • 加密小数据：如加密对称密钥或短消息。
      • 代码示例（采用 secp256k1 曲线）：
      const crypto = require('crypto');

// ECC 加密示例
function encryptECC(plaintext, publicKey) {
 const ephemeralKeyPair = crypto.generateKeyPairSync('ec', {
   namedCurve: 'secp256k1', // 使用 secp256k1 曲线
 });
 const sharedSecret = crypto.diffieHellman({
   publicKey: publicKey,
   privateKey: ephemeralKeyPair.privateKey,
 });
 const cipher = crypto.createCipheriv('aes-256-gcm', sharedSecret.slice(0, 32), Buffer.alloc(12, 0));
 let encrypted = cipher.update(plaintext, 'utf8', 'hex');
 encrypted += cipher.final('hex');
 const authTag = cipher.getAuthTag().toString('hex');
 return {
   ephemeralPublicKey: ephemeralKeyPair.publicKey.export({ type: 'spki', format: 'pem' }),
   encrypted,
   authTag,
 };
}

// ECC 解密示例
function decryptECC(encryptedData, privateKey) {
 const ephemeralPublicKey = crypto.createPublicKey(encryptedData.ephemeralPublicKey);
 const sharedSecret = crypto.diffieHellman({
   publicKey: ephemeralPublicKey,
   privateKey: privateKey,
 });
 const decipher = crypto.createDecipheriv('aes-256-gcm', sharedSecret.slice(0, 32), Buffer.alloc(12, 0));
 decipher.setAuthTag(Buffer.from(encryptedData.authTag, 'hex'));
 let decrypted = decipher.update(encryptedData.encrypted, 'hex', 'utf8');
 decrypted += decipher.final('utf8');
 return decrypted;
}

// 示例用法
const { publicKey, privateKey } = crypto.generateKeyPairSync('ec', {
 namedCurve: 'secp256k1', // 使用 secp256k1 曲线
});

const plaintext = '这是一个ECC加密测试';
const encrypted = encryptECC(plaintext, publicKey);
const decrypted = decryptECC(encrypted, privateKey);

console.log('原始数据:', plaintext);
console.log('加密结果:', encrypted);
console.log('解密结果:', decrypted);

  3. 哈希算法

    • 单向不可逆，用于数据完整性验证
    • 常见算法：SHA-256、MD5
    const crypto = require('crypto');
function hashSHA256(data) {
  return crypto.createHash('sha256').update(data).digest('hex');
}

二、进阶应用

  1. HMAC（哈希消息认证码）

    • 带密钥的哈希，防篡改
    const crypto = require('crypto');
function hmacSHA256(data, secret) {
  return crypto.createHmac('sha256', secret).update(data).digest('hex');
}

  2. 数字签名

    • 私钥签名，公钥验证
    const crypto = require('crypto');
const sign = crypto.createSign('SHA256');
sign.update('data to sign');
const signature = sign.sign(privateKey, 'hex');
// 验证签名
const verify = crypto.createVerify('SHA256');
verify.update('data to sign');
const isValid = verify.verify(publicKey, signature, 'hex');

  3. 加密模式与填充

    • 如CBC、GCM模式
    // AES-GCM加密（支持认证加密）
function encryptGCM(text, key, iv) {
  const cipher = crypto.createCipheriv('aes-256-gcm', key, iv);
  let encrypted = cipher.update(text, 'utf8', 'hex');
  encrypted += cipher.final('hex');
  const authTag = cipher.getAuthTag().toString('hex');
  return { encrypted, authTag };
}

三、密钥管理

  1. 密钥派生函数（KDF）

    • 如PBKDF2、scrypt
    const crypto = require('crypto');
function deriveKey(password, salt) {
  return crypto.pbkdf2Sync(password, salt, 100000, 32, 'sha512');
}

  2. Base64编码

    • 二进制数据转文本
    const buffer = Buffer.from('Hello Base64');
const base64 = buffer.toString('base64'); // 编码
const decoded = Buffer.from(base64, 'base64').toString(); // 解码

四、安全协议

  1. TLS/SSL

    • HTTPS通信加密
    const https = require('https');
const fs = require('fs');
const options = {
  key: fs.readFileSync('server-key.pem'),
  cert: fs.readFileSync('server-cert.pem')
};
https.createServer(options, (req, res) => {}).listen(443);

  2. JWT（JSON Web Token）

    • 基于Token的身份验证
    const jwt = require('jsonwebtoken');
const token = jwt.sign({ userId: 123 }, 'secretKey', { expiresIn: '1h' });
const decoded = jwt.verify(token, 'secretKey');

五、Node.js加密库

  • 核心模块：crypto
  • 第三方库：bcrypt（密码哈希）、node-rsa（RSA操作）
  • 区别：
    • crypto是 Node.js 内置的通用加密模块，支持多种加密算法（如 AES、SHA-256、RSA 等），适用于数据加密/解密、哈希计算、签名验证等通用场景，但不适合直接存储密码（如用 SHA-256 哈希密码易被暴力破解）。
    • bcrypt是专门用于密码哈希的库，提供了更安全的密码存储方式，如加盐、迭代计算等，能抵御彩虹表攻击，适合用户密码存储
    • node-rsa是用于 RSA 加密/解密的库，支持生成、导入、导出 RSA 密钥对，适用于数据加密/解密、数字签名、密钥交换等场景。
    • 互补使用：例如用 crypto 加密敏感数据，再用 bcrypt 存储密码。
// 使用bcrypt哈希密码
const bcrypt = require('bcrypt');
const saltRounds = 10;
bcrypt.hash('myPassword', saltRounds, (err, hash) => {
  // 存储hash到数据库
});

注意事项

  • 密钥和IV需安全存储（如环境变量、密钥管理服务）
  • 避免使用弱算法（如MD5、DES）
  • 使用高强度的随机数生成器（如crypto.randomBytes生成IV）
