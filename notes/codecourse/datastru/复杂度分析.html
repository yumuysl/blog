复杂度分析

禹穆穆2023/4/30...大约 4 分钟数据结构

目录

一、为什么需要复杂度分析

不同的数据结构对数据所需要的资源以及数据处理的效率不一，为了减少资源的损耗以及提升数据处理效率，需要事先评估当前结构或算法，所以便有了复杂度分析这样的概念，复杂度分为时间复杂度和空间复杂度，下面第二、三节有具体细节。时间复杂度和空间复杂度都底的数据结构或算法是我们的最佳选择，但这种情况出现的概率很小，更多是需要我们根据不同的情况牺牲一个而成全令一个，也就是“时间换空间”或“空间换时间”。

二、时间复杂度

  1. 时间复杂度是根据输入数据后处理数据所用时间的来判断其时间复杂度高低。一种方法是根据算法的每一步运行时间来判断时间复杂度，但在运行过程中很难去统计算法运行过程中的操作时间，而且同一算法在每台设备上运行时间也不同，所以这种方法不合理也不现实，更好的方法是第二种，即统计算法运行时间的增长趋势，随着输入的数据越多，数据处理时间增长的程度不一，根据对变量的操作判断其属于哪一种增长类型，如下：
//观察变量n，当n越大时处理效率对比越明显，体会时间增长的趋势
//O(1)，低复杂度
for(let i=0; i< 5; i++){
  console.log("时间复杂度是O(1)")
}


//O(n)
for(let i=0; i< n; i++){
  console.log("时间复杂度是O(n)")
}

//O(n^2)
for(let i=0; i< n; i++){
  for(let j=0; j< n; j++){
    console.log("时间复杂度是O(n^2)")
  }
}

//O(2^n)，指数增长非常迅速，常常不可接受，需要使用“动态规划”或“贪心算法”来替换
for(let i=0; i< n; i++){
  n *= 2；
  console.log("时间复杂度是O(2^n)")
}

...
  1. 常见时间复杂度类型由低到高排列：O(1) < O(log⁡\loglog n) < O(n) < O(n log⁡\loglog n) < O(n^2) < O(2^n) < O(n!)

三、空间复杂度

  1. 空间复杂度用于评估算法使用内存空间随着数据量变大时的增长趋势。不同算法对内存空间的使用情况也不同，使用内存空间越少的算法是最佳选择，但我们通常只关注“最差空间复杂度”，即使用空间最大这种情况，这是因为内存空间是一项硬性要求，我们必须确保在所有输入数据下都有足够的内存空间预留。我们需要判断变量占用内存空间的增长趋势属于那种类型，如下：
//O(1)
for(let i=0; i<n; i++){
  let c = 0;
}

//O(n)
function forConsole(n) {
    console.log(`递归 n = ${n}`);
    if (n === 1) return;
    forConsole(n - 1);
}

//O(n^2)
const numList = [];
for (let i = 0; i < n; i++) {
  const tmp = [];
  for (let j = 0; j < n; j++) {
        tmp.push(0);
  }
  numList.push(tmp);
}

...
  1. 常见空间复杂度类型，由低到高：O(1) < O(log⁡\loglog n) < O(n) < O(n^2) < O(2^n)

四、与后续内容衔接

基于数组 or 基于链表

  1. 基于数组的数据结构是一种静态数据结构，在内存中内存块连续，所以优点是访问快速，缺点是增加、删除、更新等操作不方便，虽然一些高级语言可以通过数组方法增删查改数组内的数据，但这种数组是在底层通过处理静态数组而得到的动态数组，所以这里的数组都是静态的。基于数组可以实现如下数据结构：栈、队列、哈希表、树、堆、图、矩阵、张量（维度 的数组）等。

  2. 基于链表的数据结构是一种动态数据结构，在内存中内存块不连续，位置分布不规律，所以其优点是增加、删除、更新数据比较方便，缺点是对数据的访问比较慢，都要从头节点一步一步找。基于链表可以实现如下数据结构：栈、队列、哈希表、树、堆、图等；

  3. 在适合的场景使用合适的数据结构，而基于数组还是基于链表也是需要在实现落地过程中酌情考虑的，如果对数据的访问过多可以基于数组，而对于数据的增加、删除、更改过多可以基于链表，提升算法整体效率。
