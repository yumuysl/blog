禹穆穆2023/4/8...大约 9 分钟前端笔记

git使用笔记

一、流程

（一）大体流程

  1. 主要在本地工作区和github/gitee项目（远程主机）之间相互作用
  2. 从本地到远程主机的操作：添加到暂存区 --> 提交到仓库区 --> 推送到远程主机
  3. 从远程主机到本地的操作：克隆 / 拉取

（二）初学时注意的几个点

  1. 本地三个区：工作区（就是平时存放代码的地方），暂存区（.git文件夹中index），仓库区（本地仓库，就是.git文件夹）
  2. git可以建立分支和标签，有本地分支和远程分支，远程标签和本地标签
  3. git可以在推送到远程主机之前可以撤回
  4. 克隆和拉取的区别：克隆是把之前本地没有的整个项目从0到1复制到本地，而拉取是把远程主机更新的内容（已经克隆过了）拉取到本地

二、相关命令

（一）配置

  1. 下载安装后打开终端
  2. 让git记住你：全局配置用户名和email地址
git config --global user.name "你的github账号"
git config --global user.email "你的github邮箱"

//查看配置
git config --global --list

//其他-设置代理和取消代理
git config --global http.proxy http://127.0.0.1:7891
git config --global https.proxy http://127.0.0.1:7891
git config --global --unset http.proxy
git config --global --unset https.proxy
  1. 配置SSH公钥：在c盘C:\Users\用户名.ssh文件夹下，打开git终端，输入 ssh-keygen -t rsa 回车，出现两个文件，打开pub的文件（公钥），将里面的内容复制到github设置面包中的ssh配置里。
  2. 【选】当前仓库配置：
git config --local user.name "你的github账号"
git config --local user.email "你的github邮箱"

//查看配置
git config --local --list
  1. 【选】删除配置
//删除全局配置
git config --unset --global 配置项

//删除当前仓库配置
git config --unset --local 配置项

（二）添加、删除、提交、查看、撤销

//新建文件夹后，初始化为git代码库
git init 项目名称

//克隆到本地 
git clone url地址 dir本地目录(克隆到哪)

//拉取到本地
git pull url地址
git fetch 

/************************************** 添加 **************************************/
//添加文件
git add 文件1  文件2

//添加当前目录的所有文件到暂存区
git add .

/************************************** 删除 **************************************/
//删除工作区文件且将这次删除放入暂存区
git rm 文件1 文件2

//停止追踪指定文件，但该文件会保留在工作区
git rm --cached 文件

/************************************** 提交 **************************************/
//从暂存区提交到仓库区
git commit -m '文件说明'

//提交时显示所有的diff信息
git commit -v

//commit id，x通常是很长的一个字符串，表示每次提交的id，在git log可以查看相关id
git commit x...

/************************************** 查看 **************************************/
//查看有变更的文件
git status

//显示当前分支的版本历史
git log

//显示commit历史，以及每次commit发生变更的文件
git log  --stat

//显示所有提交过的用户，按提交次数排序
git shortlog -sn

//显示指定文件是谁在什么时间修改过
git  blame  文件1

//显示暂存区和工作区的差异
git  diff

//查看今天写了多少代码
git  diff --shortstat "{0 day ago}"

//显示某次提交的元数据和内容变化
git show

//显示当前分支的最近几次提交
git reflog

/************************************** 撤销 **************************************/
//恢复暂存区的指定文件到工作区
git checkout 文件1

//恢复暂存区的所有文件到工作区
git checkout .

//重置暂存区的指定文件，与上一次commit保持一致，但工作区不变
git reset 文件1

//重置暂存区与工作区，与上一次commit保持一致（一个 ^ 表示退回到上一次）
git reset --hard^

//重置当前分支的指针为指定的commit，同时重置暂存区，但工作区不变
git reset commit_id

//重置当前分支的HEAD为指定commit，同时重置暂存区和工作区，与指定commit一致
git reset --hard commit_id

//重置当前HEAD为指定commit，但保持暂存区和工作区不变
git reset --keep commit_id

//新建一个commit，用来撤销指定commit
//后者的所有变化都将被前者抵消，并且应用到当前分支
git revert commit_id

//暂时将未提交的变化移除，稍后再移入
git stash
git stash pop

（三）本地和远程仓库及其同步

//把仓库区文件同步到远程仓库，注意分支是先本地后远程，和pull相反
git push  远程主机名   本地分支名 ： 远程分支名

//如果当前分支只有一个追踪分支，那么主机名可以省略
git push

//如果当前分支与多个主机存在追踪关系，首次使用分支时用-u
//这里的origin是指默认的远程主机名，master指远程仓库和本地仓库的master分支
git push -u origin master

//将本地的master分支同步到远程origin主机的master分支，如果后者不存在，则会被新建。如果省略本地分支名，则表示删除指定的远程分支，因为这等同于推送一个空的本地分支到远程分支。
git push origin master

//删除origin主机的master分支
git push origin :master  //等同于 git push origin --delete master

//将本地所有分支都同步到远程主机
git push --all origin

//使用-tags同步标签
git push origin --tags


//列出所有远程主机
git remote -v

//克隆时，远程主机默认的origin，需要改用其他主机名时，用-o指定
git clone -o 主机名 url地址

//添加远程主机
git remote add 主机名 url地址

//删除远程主机
git remote rm 主机名

//更改远程主机名
git remote rename 原主机名 新主机名

（四）分支和标签

/************************************** 分支 **************************************/
//列出所有本地分支
git branch

//列出所有远程分支
git branch -r

//新建一个分支但不切换分支
git branch 分支名

//切换到指定分支并更新工作区
git checkout 分支名

//创建并切换到名为 feature/new-login 的分支
git checkout -b feature/new-login

//合并指定分支到当前分支
git merge 分支名

//删除本地分支
git branch -d 分支名

//删除远程分支
git push origin --delete 分支名
git branch -dr 主机名/分支名

/************************************** 标签 **************************************/
//列出所有tag
git tag

//在当前的commit新建一个tag
git tag 标签名

//在指定commit新建一个tag
git tag 标签名 commit_id

//删除本地tag
git tag -d 标签名

//删除远程tag
git push origin :refs/tags/标签名

//查看tag信息
git show 标签名

//提交指定tag
git push 主机名 标签名

//提交所有tag
git push 主机名 --tags

三、提交规范

  • 核心规范，要求每个提交信息都带有一个特殊类型的前缀，结构如下：
<类型>[可选 作用域]: <描述>

[可选 正文]

[可选 页脚]
  • 类型：
    • feat: ​新增功能​（feature）。提交会使得minor版本号增加（语义化版本中的次版本号）
    • fix: ​修复bug。提交会使得patch版本号增加（语义化版本中的修订号）
    • ​docs: ​文档更新​（documentation）
    • style: ​代码格式调整​（不影响代码逻辑的变动
    • refactor: ​代码重构​（既不是新增功能，也不是修复bug）。对现有代码的结构、设计进行优化，但不改变外部行为
    • perf: ​性能优化​（performance）。提升性能的代码更改
    • ​test: ​测试相关。增加或修改测试用例（单元测试、集成测试等）。
    • build: ​构建系统或外部依赖更改。影响构建工具或外部依赖的更改（例如：gulp, broccoli, npm, webpack, rollup等配置）
    • ci: ​CI配置文件和脚本更改。持续集成相关的更改（例如：Travis, Circle, GitHub Actions, GitLab CI等配置）
    • chore: ​其他杂项修改。不属于以上类型的更改，例如构建流程辅助工具、库的更改
    • revert: ​回滚先前的提交。
  • 作用域 (Scope) - 可选：作用域用于说明提交影响的范围，通常是代码所在的模块或文件名。例如 feat(api):, fix(login):, refactor(router):它应该被括号 ()包裹。
  • 描述 (Description)：描述是提交信息的简短总结，必须紧跟类型和作用域之后。
    • 使用祈使句，现在时态​：例如change而不是changed或changes
    • 首字母不要大写
    • ​结尾不要加句号
  • 正文 (Body) - 可选：在描述之后空一行，可以编写更详细的正文，说明提交的动机、与之前行为的对比。正文同样使用祈使句、现在时态
  • 页脚 (Footer) - 可选：页脚通常用于放置一些元信息。
    • 不兼容的变更 (BREAKING CHANGE)​: 如果本次提交包含与之前版本不兼容的API变更，必须以BREAKING CHANGE:开头，后跟描述。
    • 关闭Issue: 例如 Closes #123, Fixes #456, #789

相关问题记录

添加记录时由于文件内容中换行问题引发的警告

  • 问题：warning: in the working copy of 'vite.config.js', LF will be replaced by CRLF the next time Git touches it
  • 原因：格式化与多余空白字符，在跨平台情况下由于编辑器不同文件尾行的换行符在windows下被替换了。
  • 解决方案：
    • 执行git config --global core.autocrlf true

push时出现无法连接到github

  • 问题1：fatal: unable to access 'URL_ADDRESS- 问题：fatal: unable to access 'https://github.com/': SSL certificate problem: unable to get local issuer certificate
    • 原因：证书问题
    • 解决方案：
      • 执行git config --global http.sslVerify "false"
      • 执行git config --global --unset http.proxy
  • 问题2：ssh: connect to host github.com port 22: Connection timed out. fatal: Could not read from remote repository.
    • 原因：仓库确实存在，只能是22端口问题了，可能是防火墙或者其他应用占用了22端口
    • 解决方案：进入.ssh文件夹，打开config文件，添加如下内容(对于gitee同理)：
      Host github.com
User git
Hostname ssh.github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa
port 443

github更新了username，如何提交本地仓库代码

  1. 更新本地远程连接：git remote set-url origin git@github.com:新用户名/仓库名.git
  2. 更新全局配置：git config --list user.name '新用户名'
  3. 更新本地配置：git config user.name '新用户名'
