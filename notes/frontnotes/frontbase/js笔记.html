一、变量及其类型

变量及其作用域

  • var在函数外声明的变量作用域是全局，在函数内声明的作用域在函数内（局部）
  • 在相同作用域或块级作用域中，let/var不能重置let声明的变量，let也不能重置var声明的变量
  • const 用于声明一个或多个常量，声明时必须进行初始化，且初始化后值不可再修改，但是可以改变对象或者数组内部的值

数据类型及其转换

  • 基本类型： String，Number，Boolean，Null，Undefined，Symbol，BigInt
  • 引用类型：Object，Array，Function，RegExp(正则)，Date

    • typeof：NaN数据类型是number，数组、日期、null数据类型是object（历史遗留bug），未定义的数字类型为undefined

    • Array.isArray()：最可靠检查是否为数组的方法
  • 类型转换：
    • 数字和布尔转字符串：String()或者x.toString()
    • 日期转字符串：Date()，或者obj.toString()

六、运算

算术运算符

加法 +: 用于相加两个值。 减法 -: 用于相减两个值。 乘法 *: 用于相乘两个值。 除法 /: 用于相除两个值。 取余 %: 返回除法运算后的余数。 递增 ++: 将操作数的值增加 1。 递减 --: 将操作数的值减少 1。

比较运算符

  • 等于 ==: 检查两个操作数是否相等。
  • 全等于 ===: 检查两个操作数是否严格相等（包括类型和数值）。
  • 不等于 !=: 检查两个操作数是否不相等。
  • 不全等于 !==: 检查两个操作数是否不严格相等。
  • 大于 >: 检查左操作数是否大于右操作数。
  • 小于 <: 检查左操作数是否小于右操作数。
  • 大于等于 >=: 检查左操作数是否大于或等于右操作数。
  • 小于等于 <=: 检查左操作数是否小于或等于右操作数。

逻辑运算符

  • 与 &&: 当且仅当所有操作数都为真时返回真。
  • 或 ||: 当至少有一个操作数为真时返回真。
  • 非 !: 用于翻转操作数的逻辑状态。

位运算符

  • 与 &: 对两个数的每一位执行与操作。
  • 或 |: 对两个数的每一位执行或操作。
  • 异或 ^: 对两个数的每一位执行异或操作。
  • 非 ~: 反转操作数的位。
  • 左移 <<: 将操作数的位向左移动指定的次数。
  • 右移 >>: 将操作数的位向右移动指定的次数。
  • 无符号右移 >>>: 将操作数的位向右移动指定的次数，且移动过程中使用 0 填充。

其他运算符

  • 赋值 =: 将右操作数的值赋给左操作数。
  • 条件 ?:: 根据条件返回值。
  • 逗号 ,: 用于对两个表达式进行求值，并返回最后一个表达式的值。
  • 连接选择： ?. ，用于获取对象属性值，但如果没有属性不报错，返回undefined

高阶函数

  • 高阶函数是函数式编程的核心概念之一
  • 满足下面其中一个条件就是高阶函数：
    • 函数参数是一个函数，比如数组的map、reduce等方法，
    • 返回值为一个函数：比如bind方法
  • 每个js文件中都有高阶函数，但这么多函数方法都是高阶函数，高阶函数有什么用？
    • 用于：运算的缺失和延续
    • 如何理解：当在函数中存在某个运算缺失时，需要传递一个函数；当在不确定何时运行函数时返回一个函数然其他函数决定何时运算
  • 使用和编写
    • 使用：常见的map、filter、reduce等内置高阶函数的使用
    • 编写：自己定义高阶函数，常用于公共函数、公共模块，重要且值钱

内置函数

  • 如filter()、map()等函数

闭包

  • 闭包指函数可以访问并操作其词法作用域以外的变量
function outerFunction() {
  var outerVariable = 'I am outside!';

  function innerFunction() {
    console.log(outerVariable); // 内部函数可以访问外部函数的变量
  }

  return innerFunction;
}

var closure = outerFunction(); // 调用外部函数，返回内部函数
closure(); // 输出：I am outside!

闭包的主要特点是：

  • 内部函数可以访问外部函数的变量和参数。
  • 外部函数的执行上下文和作用域链会被保存在闭包中。
  • 外部函数的变量和参数不会被垃圾回收机制回收，直到闭包存在。

类

  1. 类的由来
    • ES5中面向对象的写法与主流语言相差很大，所以ES6中添加了class关键字，引入类的概念
    • 类的数据类型就是函数，类本身就是指向构造函数
    • 类的所有方法都定义在类的prototype属性上面
    • 在类的实例上面调用方法，其实就是调用原型上的方法
    • Object.assign()方法可以很方便的一次向类添加多个方法
    • 当多个事物用共同的属性、行为方法时，可以考虑面向对象的处理方式
class Point{

  }

Object.assign(Point.prototype, {

  1. constructor()方法

    • 一个类必须有一个constructor()方法，如果没有显式定义，JavaScript引擎会自动添加constructor()方法
    • constructor()方法默认返回实例对象（即this）

  2. 类的实例

    • 类的属性和方法，除非显式定义在其本身（即定义在this对象上），否则都是定义在原型上（即定义在class上）。

  3. 静态方法和静态属性

    • 什么是静态方法：类的实例都会继承类中定义方法，但如果在一个方法前加上static关键字，表示该方法不会被实例继承，而是直接通过类调用。（但是会被子类继承）
    • 什么是静态属性：指的是Class本身的属性，即Class.proName，而不是定义在实例对象(this)上的属性
    • 总结：和具体实例无关的属性或方法属于这个类的静态对象、静态方法，比如兔子类有个总数，这个总数一定要创建一个兔子对象才能使用的话就走不通了，当无兔子对象时总数为0，比如Number.isNaN()，isNaN是属于Number类的静态方法，而不是某个数字对象的方法，不是要创建一个数字去判断另一个数字。

十二、Iterator

什么是Iterator

  • 遍历器（Iterator）就是这样一种机制。它是一种接口，为各种不同的数据结构提供统一的访问机制。任何数据结构只要部署 Iterator 接口，就可以完成遍历操作
  • 作用
    • 各种数据结构，提供一个统一的、简便的访问接口
    • 使得数据结构的成员能够按某种次序排列
    • ES6 创造了一种新的遍历命令for...of循环，Iterator 接口主要供for...of 消费
  • 原理
    • 一个可迭代对象必须在其[Symbol.iterator]​属性上部署一个方法。调用这个方法会返回一个遍历器对象（Iterator），这个对象核心是拥有一个 next()方法，每次调用next方法，都会返回包含value和done两个属性的对象
    • 遍历过程是：创建一个指针对象指向数据结构起始位置，通过不断调用next()移动指针，直到遍历完成
  • 案例
// 示例：获取数组的遍历器并手动调用 next()
let arr = ['a', 'b', 'c'];
let iterator = arr[Symbol.iterator](); // 获取遍历器对象

console.log(iterator.next()); // { value: 'a', done: false }
console.log(iterator.next()); // { value: 'b', done: false }
console.log(iterator.next()); // { value: 'c', done: false }
console.log(iterator.next()); // { value: undefined, done: true }

基本用法

  • 原生可迭代对象：Array、Set、Map、String、DOM集合等js内置数据结构默认实现Iterator接口，可以直接用for...of遍历
// 使用 for...of 遍历数组（背后自动调用 Iterator 接口）
for (let item of ['a', 'b', 'c']) {
  console.log(item); // 依次输出 'a', 'b', 'c'
}

// 遍历 Set
let mySet = new Set([1, 2, 3]);
for (let value of mySet) {
  console.log(value); // 依次输出 1, 2, 3
}

// 遍历字符串
for (let char of 'hello') {
  console.log(char); // 依次输出 'h', 'e', 'l', 'l', 'o'
}
  • 让普通对象可迭代
    • 需要手动在其 [Symbol.iterator]属性上部署遍历器生成方法
// 示例：为自定义对象添加 Iterator 接口
let myObject = {
  data: ['hello', 'world'],
  [Symbol.iterator]() { // 部署接口
    let index = 0;
    let self = this;
    return {
      next() {
        if (index < self.data.length) {
          return { value: self.data[index++], done: false };
        } else {
          return { value: undefined, done: true };
        }
      }
    };
  }
};

for (let item of myObject) {
  console.log(item); // 依次输出 'hello', 'world'
}
  • 调用 Iterator 接口的场合
    • for...of
    • 解构赋值：let [x, y] = new Set(['a', 'b']); // x = 'a', y = 'b'
    • 扩展运算符：...
    • yield*：
    function* gen() {
  yield* [1, 2, 3]; // 会调用数组的 Iterator 接口
}

进阶用法

  • 创建无限迭代器或用于自定义数据结构
// 示例：创建一个生成无限序列的迭代器（如递增ID）
function idMaker() {
  let index = 0;
  return {
    next() {
      return { value: index++, done: false };
    }
  };
}

let it = idMaker();
console.log(it.next().value); // 0
console.log(it.next().value); // 1
// ... 可以无限调用下去
  • 遍历器对象的 return()，throw()
    • 遍历器除了next()方法外还有return()和throw()方法
    • return()：在遍历完前需要提前退出遍历，常用于清理或释放资源
    function readLinesSync(file) {
  return {
    [Symbol.iterator]() {
      return {
        next() {
          return { done: false };
        },
        return() {
          file.close();
          return { done: true };
        }
      };
    },
  };
}
    • throw()：throw()方法主要是配合 Generator 函数使用

Promise

  • 三种状态：pending（进行中）、fulfilled（已成功）和rejected（已失败）
  • resolve：将状态从pending变为fulfilled，状态不可逆，参数是成功的结果值，触发.then()中第一个回调函数
  • reject：将状态从pending变为rejected，状态不可逆，参数通常是一个Error对象，触发.catch()或者.then()中第二个回调函数
//创建
let yourPromise = new Promise((resolve, reject) => { ... })

//使用
yourPromise.then((data) => {...})
  .catch((error) = { ... })

//链式调用
promise().then(data1).then(data2).catch(err).finally(data3)

//调用 resolve时传入的参数，会成为后续 .then()方法的输入值
new Promise((resolve, reject) => {
    setTimeout(() => {
        resolve('操作成功！'); // 传递字符串
    }, 1000);
}).then(result => {
    console.log(result); // 输出："操作成功！"
});

//可以直接返回resolve(value)，阻止执行器函数后续的代码继续运行，可避免意外的副作用
new Promise((resolve, reject) => {
    if (/* 成功条件 */) {
        return resolve('成功'); // 调用resolve并立即返回
        console.log('这行代码不会执行'); // 被return阻断
    }
    // ... 其他逻辑
});

//Promise.all()和Prpmise.race()
const promise1 = fetchData1();
const promise2 = fetchData2();

Promise.all([promise1, promise2])
  .then((results) => {
    console.log('所有操作完成', results);
  });

Promise.race([promise1, promise2])
  .then((result) => {
    console.log('任意一个操作完成', result);
  });
  • 其他Promise方法：

    • resolve()：将其他对象转为promise对象

// 等价于
new Promise(resolve => resolve('foo'))
    • reject()：返回一个状态为rejected的对象

p.then(null, function (s) {
console.log(s)

async

  • async返回的是一个Promise对象，当async内部的异步操作完成后才会执行后面的then()
  • await命令后面是一个Promise对象，返回该对象的结果，如果不是Promise对象会直接返回对应的值
  • 用async标记此步有异步操作，当遇到await会优先返回，等到所有异步操作完成后，执行下一步语句
  • 错误处理：如果await后面的异步操作出错，那么等同于async函数返回的 Promise 对象被reject

  });

JS基本功：问题考察

1、0.1 + 0.2 ！= 0.3，请说明原因

  • 将0.2转为二进制后是一个不断循环的二进制数，但内存容量是有限的，所以最后一位数进一表示结束，导致最终结果大于0.2
  • 将0.3转为二进制后同上，但最后一位进一是0，最终结果小于0.3
  • 而0.3 + 0.2 == 0.5成立，是因为上述的误差被抵消
//转为二进制
0.2.toString(2)

2、 parseInt执行结果是多少？

  • 先看下面的代码
console.log(["1","2","3"].map(parseInt))
//[ 1, NaN, NaN ]
  • 为什么打印出来的不是[1,2,3]?
    • 首先map用法：map(itme, index, arr)，参数分别是当前循环元素，当前循环索引，数组本身；
    • parseInt用法：parseInt(string, radix)，参数分别是元素字符串、进制数，当radix没有时根据进制规则自动判断string是几进制（ox开头为16进制，0开头为8进制，其他为10进制）
    • 上述代码中，第一次循环时其实执行的是parseInt("1", 0)，第二次循环时执行的是parseInt("2", 1)，第三次循环执行的是parseInt("3", 2)，后两个循环由于找不到该进制下的数，所以返回NaN

3、 变量a是一个对象类型，什么情况下a==1 && a==2 && a==3成立

  • 本质考察的是类型隐式转换，当字符串和数字类型比较时全都转为数字类型再判断；当布尔类型和数字类型比较时全都转为数字类型；而当对象类型和数字类型比较时，对象类型转为原始值
  • 对象有valueOf方法和toString()方法，这两个方法将对象转为原始值，但优先调用valueOf方法，当变量为数组时，valueOf不会把数组对象转为原始值，但toString()方法会调用join()方法将数组转为字符串，然后比较`
  • 当a为如下对象时，上述逻辑成立
let a = {
  value: 1,
  valueOf: function(){
    return this.value++
  }
}

console.log( a== 1 && a==2 && a==3) // true

4、 let、var和const区别

1、作用域

  • var声明的变量其作用域是全局作用域或函数作用域
  • let和const生命的变量其作用域是块作用域（用花括号括起来的区域）

2、更新和声明变量

  • var变量可以更新和重新声明
  • let变量可以更新但不能重新声明
  • const变量既不能更新也不能重新声明

3、变量提升 他们都被提升到作用域顶部，但var变量会被初始化为undefined，let和const不会被初始化，所以后面两个在声明前执行会报错，而var则显示undefined

5、闭包

  • 闭包的三个特点：
    • 每当一个函数在另一个函数中被创建时，会产生闭包
    • 闭包可以访问外部函数的变量，即使外部函数已经返回了
    • 闭包保存外部函数变量的引用，而不是实际的值

6、NaN表示什么意思？如何判断NaN的值

  • NaN全称"Not a Number"，表示不是一个数字
  • NaN和任何值相比都是false，包括其自身
  • NaN检测：isNaN判断是否为数字，包含类型转换；Number.isNaN也可以判断是否为数字，但不包含类型转换，比如Number.isNaN("23")返回的是false

7、void 0和undefined区别

  • undefined可以被改变，当声明一个变量名为undefined的变量，然后赋值，这时undefined原本的值被改变了，当后续的代码用undefined后会出现bug
  • void 0可以安全的获取undefined

8、bind、apply、call的区别和应用场景

下面代码两个打印结果分别是什么？

var name = "李四"
let obj = {
  name: "张三",
  getName: function(){
    console.log(this.name)
  }
}

obj.getName();
setTimeout(obj.getName, 10)

第一个打印的是"张三"，第二个在Node.js环境下打印的是undefined，在浏览器环境下打印的是李四，那为什么打印的结果不同呢？这就涉及到this绑定规则。

  1. this指向它的调用者，在obj.getName中，this指向obj，所以打印的是张三，但在setTimeout(obj.getName, 10)语句中，传给setTimeout的参数其实是函数的引用，并没有保存函数的对象obj，所以this指向执行上下文，但我们并没有绑定this的执行上下文，这时this指向全局对象，在浏览器环境中全局对象为window，在Node.js环境中，全局对象为global，所以分别打印的是undefined和李四。
  2. 如何修改this的指向？
    • 通过apply绑定this：apply(obj)中绑定this给obj，也就是让obj调用this，所以上面代码可以改成这样：
      var name = "李四"
let obj = {
  name: "张三",
  getName: function(){
    console.log(this.name)
  }
}

obj.getName();

setTimeout(function(){
  obj.getName.apply(obj)
}, 10)
    • 通过call绑定this：此方法和apply方法基本一致，但当getName有参数需要传递时，apply第二个参数是一个数组，数组内元素分别是getName的参数，而call方法除第一个参数外，其他都是和getName的参数一一对应的参数，如下：
      //apply
var name = "李四"
let obj = {
  name: "张三",
  getName: function(age, career){
    console.log(this.name, age, career)
  }
}

obj.getName();

setTimeout(function(){
  obj.getName.apply(obj, [20, "医生"])
}, 10)

//call
var name = "李四"
let obj = {
  name: "张三",
  getName: function(age, career){
    console.log(this.name, age, career)
  }
}

obj.getName();

setTimeout(function(){
  obj.getName.call(obj, 20, "医生")
}, 10)
    • 通过bind方法绑定this：此方法和上面两个方法最大区别是bind方法返回的是一个新的函数，只有执行新函数时才会绑定this，另外传递的参数也可以在新的函数中传入
      var name = "李四"
let obj = {
  name: "张三",
  getName: function(age, career){
    console.log(this.name, age, career)
  }
}

setTimeout(function(){
  let newFunc = obj.getName.bind(obj, 20, "医生")
  newFunc();

  //或者
  let newFunc = obj.getName.bind(obj)
  newFunc( 20, "医生");

}, 10)

9、什么是柯里化？如何实现柯里化？

柯里化（Currying）是一种函数式编程的概念，它指的是将接受多个参数的函数转换为一系列接受一个参数的函数的过程。柯里化的主要作用是让函数更灵活，能够更方便地复用和组合。

柯里化的使用场景包括：

  1. 参数复用：柯里化可以让我们事先固定一部分参数，后续调用时只需传入剩余的部分参数。
  2. 延迟执行：柯里化可以让函数在接收到足够的参数后再执行，从而实现延迟执行的效果。
  3. 函数组合：柯里化可以方便地将多个函数组合在一起，形成新的函数，实现更复杂的功能。

如何实现柯里化：

  • 实现思路： 如果参数达到length，直接计算结果，否则返回一个新的函数
  • 代码实现
    function sum(a, b, c){
  console.log(a + b + c)
}

function carry(fn){
  return function carried(...args){
    if(args.length >= fn.length){
      return fn.apply(this, args);
    }else{
      return function(...args2){
        return carried.apply(this, args.concat(args2))
      }
    }
  }
}

let _sum = carry(sum)
let functionA = _sum(1)
let functionB = functionA(2)
functionB(3)

    if (typeof (cell) === 'number') return String(cell);

function formatDateToCustomString(date) {
    // 获取年份、月份、日期等信息
    const year = date.getFullYear();
    const month = String(date.getMonth() + 1).padStart(2, '0'); // 月份从0开始，所以要加1
    const day = String(date.getDate()).padStart(2, '0');

14. 面向对象、面向函数和面向过程的区别

  • 指的是思维方式，而不是说创建了个函数就是面向函数编程，是思维而不是具体的代码
  • 面向过程：类似流程水帐，第一步做什么，第二步做什么，第三步做什么....，可以处理很多简单的东西，但当复杂度上来后
  • 面向函数：每个函数都是纯函数，关注的运算关系，比如把某个函数返回的结果作为参数传给下一个函数，都是计算以及计算和计算之间的关系
  • 面向对象：关注的是整个系统中有哪些东西或事物，东西与东西之间的关系是怎么样的，把共性拿处理放到类中，处理事物的行为和状态
