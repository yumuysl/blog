安装和运行

安装

  • 全局安装TS：pnpm install -g typescript
  • 验证查看版本：tsc -v

编译运行ts

  • 写一份demo，例如demo.ts：
function greet(name: string): string {
  return `Hello, ${name}!`;
}
console.log(greet("World"));
  • 编译：tsc demo.ts，会生成demo.js
  • 运行：node demo.js

高效方法

安装插件
  • 安装插件运行，如ts-node
npm install -g ts-node
# 需要package.json文件，但建议直接生成tsconfig.json文件
tsc init  # 生成tsconfig.json文件
ts-node app.ts  # 直接运行，不生成 .js 文件
  • 这种方式适合小项目或快速测试
配置 tsconfig.json（复杂项目必备）
  • 生成配置文件：tsc --init，自动生成配置文件
  • 示例配置：
{
  "compilerOptions": {
    "target": "ES6",      // 编译为 ES6 代码
    "outDir": "./dist",   // 输出目录
    "strict": true        // 启用严格类型检查
  },
  "include": ["src/​**​/*"] // 编译 src 目录下的文件
}
  • 编译整个项目：tsc
  • 启用监视模式：tsc --watch，可在修改文件后自动更新

TS类型声明及类型系统

  • 采用变量:类型的形式声明变量
let str : string = '字符串'
  • 如果声明的类型和值的类型不同会报错
  • 类型推断（隐式声明）：类型显示声明并不是必须的，当不声明类型是ts会自动推断类型，如let str = '字符串'，这是隐式声明了变量类型，当后续str变量的值不为字符串时会报错
  • 基本类型：number, string, boolean, null, undefined, symbol, bigint
  • 包装类型： Boolean、String、Number、BigInt、Symbol
  • 特殊类型：any（禁用类型检查）、unknown（安全版 any）、void（无返回值）、never（永不返回的函数）
  • 值类型：
let x: "hello";

x = "hello"; // 正确
x = "world"; // 报错
  • 联合类型：
let x: string | number;

x = 123; // 正确
x = "abc"; // 正确
  • 交叉类型：let x: number & string;，主要用途是表示对象的合成
let obj: { foo: string } & { bar: string };

obj = {
  foo: "hello",
  bar: "world",
};
  • type命令：别名
type Age = number;
let age: Age = 55;
  • typeof命令：TypeScript 将typeof运算符移植到了类型运算，它的操作数依然是一个值，但是返回的不是字符串，而是该值的 TypeScript 类型。
    • unknown：不能直接调用unknown类型变量及其方法和属性，但可以经过typeof运算以后，能够确定实际类型实现“类型缩小”，即将一个不确定的类型缩小为更明确的类型。
const a = { x: 0 };
type T0 = typeof a; // { x: number }
type T1 = typeof a.x; // number

let a: unknown = 1;
a + 1; // 报错
a === 1; // 正确
if (typeof a === "number") {  //类型缩小
  let r = a + 10; // 正确
}
  • 块级类型声明：TypeScript 支持块级类型声明，即类型可以声明在代码块（用大括号表示）里面，并且只在当前代码块有效。
if (true) {
  type T = number;
  let v: T = 5;
} else {
  type T = string;
  let v: T = "hello";
}
  • 类型的兼容
type T = number | string;
let a: number = 1;
let b: T = a;

数组

  • TypeScript 数组有一个根本特征：所有成员的类型必须相同，但是成员数量是不确定的，可以是无限数量的成员，也可以是零成员。
  • 两种写法：
    • let arr: number[] = [1, 2, 3]; 或 let arr: (number | string)[];
    • let arr: Array<number> = [1, 2, 3]; 或 let arr: Array<number | string>;
  • 类型推断
    • 如果变量的初始值是空数组，那么 TypeScript 会推断数组类型是any[]，之后为这个数组赋值时，TypeScript 会自动更新类型推断
    const arr = [];
arr; // 推断为 any[]

arr.push(123);
arr; // 推断类型为 number[]

arr.push("abc");
arr; // 推断类型为 (string|number)[]
    • 类型推断的自动更新只发生初始值为空数组的情况。如果初始值不是空数组，类型推断就不会更新。
    // 推断类型为 number[]
const arr = [123];
arr.push("abc"); // 报错
  • const断言
    • js中通过const声明的数组可以改变其子元素的值，TypeScript 允许声明只读数组，方法是在数组类型前面加上readonly关键字
    const arr: readonly number[] = [0, 1];

arr[1] = 2; // 报错
arr.push(3); // 报错
delete arr[0]; // 报错
    • readonly关键字不能与数组的泛型写法一起使用，要使用ReadonlyArray或Readonly
    // 报错
const arr: readonly Array<number> = [0, 1];
//正确
const a1: ReadonlyArray<number> = [0, 1];
const a2: Readonly<number[]> = [0, 1];

元组

  • 元组表示成员类型可以自由设置的数组，即数组的各个成员的类型可以不同。
  • 元组必须明确声明每个成员的类型，且成员类型是写在方括号里面（[number]）
const s: [string, string, boolean] = ["a", "b", true];
  • 元组成员的类型可以添加问号后缀（?），表示该成员是可选的，并且问号只能用于元组的尾部成员，也就是说，所有可选成员必须在必选成员之后
let a: [number, number?] = [1];
  • 使用扩展运算符...，可以表示不限成员数量的元组
type NamedNums = [string, ...number[]];
const a: NamedNums = ["A", 1, 2];
const b: NamedNums = ["B", 1, 2, 3];
  • 元组可以通过方括号，读取成员类型
type Tuple = [string, number];
type Age = Tuple[0]; // string
  • 元组也可以是只读的，不允许修改，有两种写法
// 写法一
type t = readonly [number, string];

// 写法二
type t = Readonly<[number, string]>;
  • 跟数组一样，只读元组是元组的父类型。所以，元组可以替代只读元组，而只读元组不能替代元组。
type t1 = readonly [number, number];
type t2 = [number, number];

let x: t2 = [1, 2];
let y: t1 = x; // 正确

x = y; // 报错
  • 扩展运算符...将数组转换成一个逗号分隔的序列，这时 TypeScript 会认为这个序列的成员数量是不确定的，因为数组的成员数量是不确定的
    const arr = [1, 2];

function add(x: number, y: number) {
  // ...
}

add(...arr); // 报错
    • 为了解决这个问题，把成员数量不确定的数组，写成成员数量确定的元组，再使用扩展运算符
    const arr: [number, number] = [1, 2];
    
function add(x: number, y: number) {
  // ...
}
add(...arr); // 正确

Symbol

  • 每一个 Symbol 值都是独一无二的，与其他任何值都不相等
  • Symbol 值通过Symbol()函数生成。在 TypeScript 里面，Symbol 的类型使用symbol表示
let x: symbol = Symbol();
let y: symbol = Symbol();

x === y; // false
  • unique symbol
    • symbol无法指定某个单个的Symbol值，所以有了unique symbol
    • unique symbol表示单个值，所以这个类型的变量是不能修改值的，只能用const命令声明，不能用let声明。
    // 正确
const x: unique symbol = Symbol();
// 报错
let y: unique symbol = Symbol();
  • const命令为变量赋值 Symbol 值时，变量类型默认就是unique symbol，所以类型可以省略不写
const x: unique symbol = Symbol();
// 等同于
const x = Symbol();

对象

  • 声明对象
    • 是使用大括号表示对象，在大括号内部声明每个属性和方法的类型
    • 属性的类型可以用分号结尾，也可以用逗号结尾
const obj: {
  x: number;
  y: number;
} = { x: 1, y: 1 };
  • 声明了类型给对象赋值时，不能缺少指定的属性，也不能有多余的属性
type TestObj = {
  x: number;
  y: number;
};

const o1: TestObj = { x: 1 }; // 报错
const o2: TestObj = { x: 1, y: 1, z: 1 }; // 报错
  • 除了type命令可以为对象类型声明一个别名，还可以用interface命令，把对象类型提炼为一个接口
// 写法一
type TestObj = {
  x: number;
  y: number;
};

const obj: TestObj = { x: 1, y: 1 };

// 写法二
interface TestObj {
  x: number;
  y: number;
}

const obj: TestObj = { x: 1, y: 1 };
  • 可选属性
    • 某个属性是可选的（即可以忽略），需要在属性名后面加一个问号
    • 读取可选属性之前，必须检查一下是否为undefined
const user: {
  firstName: string;
  lastName?: string;
} = { firstName: "Foo" };

if (user.lastName !== undefined) {
  console.log(`hello ${user.firstName} ${user.lastName}`);
}
  • 只读属性
    • 属性名前面加上readonly关键字，表示这个属性是只读属性，不能修改
    • 如果属性值是一个对象，readonly修饰符并不禁止修改该对象的属性，只是禁止完全替换掉该对象
interface MyInterface {
  readonly prop: number;
}

interface Home {
  readonly resident: {
    name: string;
    age: number;
  };
}

const h: Home = {
  resident: {
    name: "Vicky",
    age: 42,
  },
};

h.resident.age = 32; // 正确
h.resident = {
  name: "Kate",
  age: 23,
}; // 报错
  • 属性名的索引类型
type MyObj = {
  [property: string]: string;
};

const obj: MyObj = {
  foo: "a",
  bar: "b",
  baz: "c",
};
  • 解构赋值
const {
  id,
  name,
  price,
}: {
  id: string;
  name: string;
  price: number;
} = product;
  • 结构类型原则
    • 只要对象 B 满足 对象 A 的结构特征，TypeScript 就认为对象 B 兼容对象 A 的类型，这称为“结构类型”原则
const A = {
  x: number;
};

const B = {
  x: number;
  y: number;
};
const B = {
  x: 1,
  y: 1,
};

const A: { x: number } = B; // 正确
  • 最小可选属性规则
    • 如果一个对象的所有属性都是可选的，会触发最小可选属性规则
type Options = {
  a?: number;
  b?: number;
  c?: number;
};

const obj: Options = {
  d: 123, // 报错
};
  • 空对象
    • 空对象是 TypeScript 的一种特殊值，也是一种特殊类型
    • TypeScript 不允许动态添加属性，所以对象不能分步生成，必须生成时一次性声明所有属性
    • 如果确实需要分步声明，一个比较好的方法是，使用扩展运算符...合成一个新对象
// 错误
const pt = {};
pt.x = 3;
pt.y = 4;

// 正确
const pt = {
  x: 3,
  y: 4,
};

//正确
const pt0 = {};
const pt1 = { x: 3 };
const pt2 = { y: 4 };

const pt = {
  ...pt0,
  ...pt1,
  ...pt2,
};

枚举

  • 用来定义一组相关的常量
enum Color {
  Red, // 0
  Green, // 1
  Blue, // 2
}
  • 可以使用点运算符，也可以使用方括号运算符
let c = Color.Green; // 1
// 等同于
let c = Color["Green"]; // 1
  • Enum 结构既是一种类型，也是一个值，编译后会变成 JavaScript 对象，留在代码中，因此官方建议谨慎使用 Enum 结构
//类型
let c: Color = Color.Green; // 正确
let c: number = Color.Green; // 正确

//编译后
let Color = {
  Red: 0,
  Green: 1,
  Blue: 2,
};
  • Enum成员的值
    • Enum 成员的值可以是小数，但不能是 Bigint
    • 默认不必赋值，系统会从零开始逐一递增
    • 只读，不能重新赋值
enum Color {
  Red = 90,
  Green = 0.5,
  Blue = 7n, // 报错
}
  • keyof运算符
    • keyof 运算符可以取出 Enum 结构的所有成员名，作为联合类型返回
enum MyEnum {
  A = "a",
  B = "b",
}

// 'A'|'B'
type Foo = keyof typeof MyEnum;
  • 反向映射
enum Weekdays {
  Monday = 1,
  Tuesday,
  Wednesday,
  Thursday,
  Friday,
  Saturday,
  Sunday,
}

console.log(Weekdays[3]); // Wednesday

函数及其类型

函数声明

  • 一个函数有输入和输出，要在 TypeScript 中对其进行约束，需要把输入和输出都考虑到
function sum(x: number, y: number): number {
    return x + y;
}
  • 输入多余的（或者少于要求的）参数，是不被允许的
function sum(x: number, y: number): number {
    return x + y;
}
sum(1, 2, 3);  //报错
sum(1); //报错

函数表达式

  • 在 TypeScript 的类型定义中，=>用来表示函数的定义，左边是输入类型，需要用括号括起来，右边是输出类型，在 ES6 中，=>叫做箭头函数，应用十分广泛
let mySum: (x: number, y: number) => number = function (x: number, y: number): number {
    return x + y;
};

用接口定义函数形状

interface SearchFunc {
    (source: string, subString: string): boolean;
}

let mySearch: SearchFunc;
mySearch = function(source: string, subString: string) {
    return source.search(subString) !== -1;
}

可选参数

  • 用?表示可选参数，但可选参数后面不允许再出现必需参数
function buildName(firstName: string, lastName?: string) {
    if (lastName) {
        return firstName + ' ' + lastName;
    } else {
        return firstName;
    }
}
let tomcat = buildName('Tom', 'Cat');
let tom = buildName('Tom');

默认参数

  • typeScript 会将添加了默认值的参数识别为可选参数，但此时可选参数可以在必选参数之前
function buildName(firstName: string = 'Tom', lastName: string) {
    return firstName + ' ' + lastName;
}
let tomcat = buildName('Tom', 'Cat');
let cat = buildName(undefined, 'Cat');

剩余参数

  • 使用...rest方式获取函数中的剩余参数
function push(array, ...items) {
    items.forEach(function(item) {
        array.push(item);
    });
}

let a: any[] = [];
push(a, 1, 2, 3);

重载

  • 重载允许一个函数接受不同数量或类型的参数时，作出不同的处理，比如函数可输出多个类型的值时，下列代码重复定义了多次函数 reverse，前几次都是函数定义，最后一次是函数实现
function reverse(x: number): number;
function reverse(x: string): string;
function reverse(x: number | string): number | string | void {
    if (typeof x === 'number') {
        return Number(x.toString().split('').reverse().join(''));
    } else if (typeof x === 'string') {
        return x.split('').reverse().join('');
    }
}

接口

接口定义

  • 使用interface关键字，赋值必须符合规定的类型
interface Person {
  firstName: string;
  lastName: string;
  age: number;
}

const p: Person = {
  firstName: "John",
  lastName: "Smith",
  age: 25,
};
  • 方括号运算符可以取出 interface 某个属性的类型
interface Foo {
  a: string;
}

type A = Foo["a"]; // string

五种形式

  • interface 可以表示对象的各种语法，它的成员有 5 种形式。
    • 对象属性
    • 对象的属性索引
    • 对象方法
    • 函数
    • 构造函数

对象属性

  • 加?表示属性可选
  • 加readonly表示只读
interface Point {
  x: number;
  y?: number;
}

interface A {
  readonly a: string;
}

对象的属性索引

  • 属性索引共有string、number和symbol三种类型
interface A {
  [prop: string]: number;
}

对象的方法

  • 三种写法
// 写法一
interface A {
  f(x: boolean): string;
}

// 写法二
interface B {
  f: (x: boolean) => string;
}

// 写法三
interface C {
  f: { (x: boolean): string };
}
  • 属性的方法可以重载
interface A {
  f(): number;
  f(x: boolean): boolean;
  f(x: string, y: string): string;
}

函数

interface Add {
  (x: number, y: number): number;
}

const myAdd: Add = (x, y) => x + y;

沟通函数

interface ErrorConstructor {
  new (message?: string): Error;
}

interface的继承

  • interface 可以使用extends关键字，继承其他 interface，并且允许多重继承
interface Style {
  color: string;
}

interface Shape {
  name: string;
}

interface Circle extends Style, Shape {
  radius: number;
}
  • interface 可以继承type命令定义的对象类型
type Country = {
  name: string;
  capital: string;
};

interface CountryWithPop extends Country {
  population: number;
}
  • interface继承class
    • interface 还可以继承 class，即继承该类的所有成员
class A {
  x: string = "";

  y(): boolean {
    return true;
  }
}

interface B extends A {
  z: number;
}

接口合并

  • 多个同名接口会合并为一个
interface Box {
  height: number;
  width: number;
}

interface Box {
  length: number;
}

nterface 与 type 的比较

  • 相同点
    • 都可以用来表示对象类型
  • 不同点
    • type能够表示非对象类型，而interface只能表示对象类型（包括数组、函数等）
    • interface可以继承其他类型，type不支持继承
    • 同名interface会自动合并，同名type则会报错。也就是说，TypeScript 不允许使用type多次定义同一个类型
    • interface不能包含属性映射（mapping），type可以
    • this关键字只能用于interface
    • type 可以扩展原始数据类型，interface 不行
    • interface无法表达某些复杂类型（比如交叉类型和联合类型），但是type可以

类

类的类型

  • 属性的类型
    • 类的属性可以在顶层声明，也可以在构造方法内部声明
class Point {
  x: number;
  y: number;
}
  • readonle修饰符
    • 属性名前面加上 readonly 修饰符，就表示该属性是只读的。实例对象不能修改这个属性
class A {
  readonly id = "foo";
}

const a = new A();
a.id = "bar"; // 报错
  • 方法的类型
    • 与普通函数一致，包括重载、默认参数等
class Point {
  x: number;
  y: number;

  constructor(x: number, y: number) {
    this.x = x;
    this.y = y;
  }

  add(point: Point) {
    return new Point(this.x + point.x, this.y + point.y);
  }
}
  • 存取器方法
    • 取值器（getter）和存值器（setter）两种方法
    • 如果某个属性只有get方法，没有set方法，那么该属性自动成为只读属性
    • set方法的参数类型，必须兼容get方法的返回值类型，否则报错
    • get方法与set方法的可访问性必须一致，要么都为公开方法，要么都为私有方法
class C {
  _name = "";
  get name() {
    return this._name;
  }
  set name(value) {
    this._name = value;
  }
}
  • 属性索引
    • 下面的[s:string]表示所有属性名类型为字符串的属性，它们的属性值要么是布尔值，要么是返回布尔值的函数
class MyClass {
  [s: string]: boolean | ((s: string) => boolean);

  get(s: string) {
    return this[s] as boolean;
  }
}

类的interface

  • implements 关键字
    • interface 接口或 type 别名，可以用对象的形式，为 class 指定一组检查条件。然后，类使用 implements 关键字，表示当前类满足这些外部类型条件的限制
    • implements并不能代替 class 自身的类型声明
    interface Country {
  name: string;
  capital: string;
}
// 或者
type Country = {
  name: string;

类的继承

  • 使用extends 关键字继承另一个类（又称“基类”）的所有属性和方法

  greet() {
    console.log("Hello, world!");
  }
}

class B extends A {}

const b = new B();
b.greet(); // "Hello, world!"
  • 子类可以覆盖同名方法，但子类的同名方法不能与基类的类型定义相冲突
class B extends A {
  greet(name?: string) {
    if (name === undefined) {
      super.greet();
    } else {
      console.log(`Hello, ${name}`);
    }
  }
}
  • 如果基类包括保护成员（protected修饰符），子类可以将该成员的可访问性设置为公开（public修饰符），也可以保持保护成员不变，但是不能改用私有成员（private修饰符）
class A {
  protected x: string = "";
  protected y: string = "";
  protected z: string = "";
}

class B extends A {
  // 正确
  public x: string = "";

  // 正确
  protected y: string = "";

  // 报错
  private z: string = "";
}

可访问性修饰符

  • 类的内部成员的外部可访问性，由三个可访问性修饰符access modifiers控制：public、private和protected
  • public：公开可访问
class Greeter {
  public greet() {
    console.log("hi!");
  }
}

const g = new Greeter();
g.greet();
  • private：表示私有成员，只能用在当前类的内部，类的实例和子类都不能使用该成员，但严格意义上讲，private定义的私有成员，并不是真正意义的私有成员，从ES6标准发布后，建议采用#命名私有成员
class A {
  private x: number = 0;
}

const a = new A();
a.x; // 报错

class B extends A {
  showX() {
    console.log(this.x); // 报错
  }
}

//#定义私有属性
class A {
  #x = 1;
}

const a = new A();
a["x"]; // 报错
  • protected：表示该成员是保护成员，只能在类的内部使用该成员，实例无法使用该成员，但是子类内部可以使用
class A {
  protected x = 1;
}

class B extends A {
  getX() {
    return this.x;
  }
}

const a = new A();
const b = new B();

a.x; // 报错
b.getX(); // 1

泛型

为什么需要泛型

  • 当函数输入类型和输出类型需要一致时，只能这么写
function f(arr: any[]): any {
  return arr[0];
}
  • 但上述方式反映不出参数与返回值之间的类型关系，所以出现了泛型
function getFirst<T>(arr: T[]): T {
  return arr[0];
}
getFirst<number>([1, 2, 3]);

泛型的特点

  • 带有“类型参数”
function getFirst<T>(arr: T[]): T {
  return arr[0];
}
  • 函数getFirst()的函数名后面尖括号的部分<T>，就是类型参数，参数要放在一对尖括号<>里面。本例只有一个类型参数T，可以将其理解为类型声明需要的变量，需要在调用时传入具体的参数类型。
  • 多个类型参数，则使用 T 后面的 U、V 等字母命名，各个参数之间使用逗号,分隔

泛型的写法

  • 泛型主要用在四个场合：函数、接口、类和别名

函数泛型的写法

function id<T>(arg: T): T {
  return arg;
}

//变量形式
// 写法一
let myId: <T>(arg: T) => T = id;

// 写法二
let myId: { <T>(arg: T): T } = id;

接口的泛型写法

interface Box<Type> {
  contents: Type;
}

let box: Box<string>;

类的泛型写法

class Pair<K, V> {
  key: K;
  value: V;
}

//类的继承
class A<T> {
  value: T;
}

class B extends A<any> {}

类型别名的泛型写法

type Container<T> = { value: T };

const a: Container<number> = { value: 0 };
const b: Container<string> = { value: "b" };

泛型的默认值

function getFirst<T = string>(arr: T[]): T {
  return arr[0];
}
getFirst([1, 2, 3]); // 正确

//类的泛型默认值
class Generic<T = string> {
  list: T[] = [];

  add(t: T) {
    this.list.push(t);
  }
}
const g = new Generic();

g.add(4); // 报错
g.add("hello"); // 正确

数组的泛型表示

  • 数组类型有一种表示方法是Array<T>。这就是泛型的写法，Array是 TypeScript 原生的一个类型接口，T是它的类型参数。声明数组时，需要提供T的值
let arr: Array<number> = [1, 2, 3];

类型参数的约束条件

function comp<T extends { length: number }>(a: T, b: T) {
  if (a.length >= b.length) {
    return a;
  }
  return b;
}
comp([1, 2], [1, 2, 3]); // 正确
comp("ab", "abc"); // 正确
comp(1, 2); // 报错
  • 上面示例中，T extends { length: number }就是约束条件，表示类型参数 T 必须满足{ length: number }，否则就会报错

使用注意点

  • 尽量少用泛型：泛型虽然灵活，但是会加大代码的复杂性，使其变得难读难写。一般来说，只要使用了泛型，类型声明通常都不太易读，容易写得很复杂。因此，可以不用泛型就不要用
  • 类型参数越少越好
    • 多一个类型参数，多一道替换步骤，加大复杂性。
  • 类型参数需要出现两次
  • 泛型可以嵌套：泛型的参数可以是另一方泛型
type OrNull<Type> = Type | null;

type OneOrMany<Type> = Type | Type[];

type OneOrManyOrNull<Type> = OrNull<OneOrMany<Type>>;

类型断言

i### 初始断言

  • 断言：告诉编译器此处的值是什么类型。TS一旦发现存在类型断言，就不再对该值进行类型推断，而是直接采用断言给出的类型
  • 两种写法
    • TS支持 React 的 JSX 语法（尖括号表示 HTML 元素），为了避免两者冲突，就引入了语法二，推荐使用语法二
let foo = "a"
// 语法一
let bar: T = <T>foo;

// 语法二
let bar: T = foo as T;

类型断言的条件

值的实际类型与断言的类型必须满足一个条件，类型断言要求实际的类型与断言的类型兼容，实际类型可以断言为一个更加宽泛的类型（父类型），也可以断言为一个更加精确的类型（子类型），但不能断言为一个完全无关的类型

expr as T;

as const 断言

  • 如果没有声明变量类型，let 命令声明的变量，会被类型推断为 TypeScript 内置的基本类型之一；const 命令声明的变量，则被推断为值类型常量
// 类型推断为基本类型 string
let s1 = "JavaScript";

// 类型推断为字符串 “JavaScript”
const s2 = "JavaScript";
  • as const也不能用于表达式
let s = ("Java" + "Script") as const; // 报错

非空断言

  • 对于那些可能为空的变量（即可能等于undefined或null），TypeScript 提供了非空断言，保证这些变量不会为空，写法是在变量名后面加上!
function f(x?: number | null) {
  validateNumber(x); // 自定义函数，确保 x 是数值
  console.log(x!.toFixed());
}

function validateNumber(e?: number | null) {
  if (typeof e !== "number") throw new Error("Not a number");
}
  • 非空断言会造成安全隐患，只有在确定一个表达式的值不为空时才能使用。比较保险的做法还是手动检查一下是否为空
const root = document.getElementById("root");

if (root === null) {
  throw new Error("Unable to find DOM element #root");
}

root.addEventListener("click", (e) => {
  /* ... */
});

断言函数

  • 断言函数是一种特殊函数，用于保证函数参数符合某种类型。如果函数参数达不到要求，就会抛出错误，中断程序执行；如果达到要求，就不进行任何操作，让代码按照正常流程运行
//断言函数
function isString(value) {
  if (typeof value !== "string") throw new Error("Not a string");
}
//使用断言函数
const aValue: string | number = "Hello";
isString(aValue);

模块

  • 任何包含 import 或 export 语句的文件，就是一个模块module。相应地，如果文件不包含export 语句，就是一个全局的脚本文件
  • 模块本身就是一个作用域，不属于全局作用域。模块内部的变量、函数、类只在内部可见，对于模块外部是不可见的。暴露给外部的接口，必须用 export 命令声明；如果其他文件要使用模块的接口，必须用 import 命令来输入。

import type 语句

  • 使用 import type 语句，这个语句只能输入类型，不能输入正常接口
// a.ts
export interface A {
  foo: string;
}

import { A, a } from "./a";

// b.ts
import { type A, a } from "./a";  //正确

// 正确
import type { A } from "./a";
  • export 语句也有两种方法，表示输出的是类型
type A = "a";
type B = "b";

// 方法一
export { type A, type B };

// 方法二
export type { A, B };

CommonJS 模块

  • CommonJS 是 Node.js 的专用模块格式，与 ES 模块格式不兼容
  • import = 语句：输入 CommonJS 模块
import fs = require("fs");
const code = fs.readFileSync("hello.ts", "utf8");
  • 使用import * as [接口名] from "模块文件"输入 CommonJS 模块
import * as fs from "fs";
// 等同于
import fs = require("fs");
  • export = 语句：输出 CommonJS 模块的对象，等同于 CommonJS 的module.exports对象
    • export = 语句输出的对象，只能使用import =语句加载
let obj = { foo: 123 };

export = obj;
  • 路径映射
    • 在tsconfig.json文件里面，手动指定脚本模块的路径
    • baseUrl：指定脚本模块的基准目录
    • paths：非相对路径的模块与实际脚本的映射
    • rootDirs：指定模块定位时必须查找的其他目录

  }

内置工具类型

  • ​Partial<T>: 将 T的所有属性变为可选。适用于需要处理对象部分更新的场景
interface User {
  id: number;
  name: string;
}
type PartialUser = Partial<User>;
// PartialUser 等价于 { id?: number; name?: string; }
  • Required<T>: 将 T的所有属性变为必选。与 Partial相反
interface Props {
  title?: string;
  content?: string;
}
type StrictProps = Required<Props>;
// StrictProps 等价于 { title: string; content: string; }
  • ​Readonly<T>: 将 T的所有属性变为只读。常用于确保对象常量不被修改
interface Point {
  x: number;
  y: number;
}
type ReadonlyPoint = Readonly<Point>;
// ReadonlyPoint 等价于 { readonly x: number; readonly y: number; }
  • ​Pick<T, K>: 从 T中挑选一组属性 K（K是 T的键的联合类型）来构造新类型
interface Todo {
  title: string;
  description: string;
  completed: boolean;
}
type TodoPreview = Pick<Todo, 'title' | 'completed'>;
// TodoPreview 等价于 { title: string; completed: boolean; }
  • ​Omit<T, K>: 从 T中排除一组属性 K（K是字符串字面量或字符串字面量的联合类型）来构造新类型。适用于移除敏感字段
interface User {
  id: number;
  name: string;
  password: string;
}
type SafeUser = Omit<User, 'password'>;
// SafeUser 等价于 { id: number; name: string; }
  • ​Record<K, T>: 构造一个对象类型，其属性键为 K（通常是字符串字面量的联合类型），属性值为 T。常用于映射场景
type Role = 'admin' | 'user';
type RoleDescriptions = Record<Role, string>;
// RoleDescriptions 等价于 { admin: string; user: string; }
  • ReturnType<T>: 获取函数类型 T的返回类型。适用于从已有函数推导类型，避免重复定义
function getUser() {
  return { id: 1, name: 'Alice' };
}
type UserData = ReturnType<typeof getUser>;
// UserData 等价于 { id: number; name: string; }

字段说明

  • exclude：接收一个数组，必须与include属性一起使用，用来从编译列表中去除指定的文件
{
  "include": ["**/*"],
  "exclude": ["**/*.spec.ts"]
}
  • extends：继承另一个配置文件，一个项目有多个配置，可以把共同的配置写成tsconfig.base.json，其他的配置文件继承该文件，这样便于维护和修改
{
  "extends": "../tsconfig.base.json"
}
  • files：接收一个数组，指定编译的文件列表，如果列表中有不存在的文件则报错
{
  "files": ["a.ts", "b.ts"]
}
  • include：指定所要编译的文件列表，支持下面三种通配符
    • ?：指代单个字符
    • *：指代任意字符，不含路径分隔符
    • **：指定任意目录层级
{
  "include": ["src/**/*", "tests/**/*"]
}
  • references：接收一个数组，数组成员为对象，适合一个大项目由许多小项目构成的情况，用来设置需要引用的底层项目
{
  "references": [
    { "path": "../pkg1" },
    { "path": "../pkg2/tsconfig.json" }
  ]
}
  • compileOptions：用来定制编译行为，如果这个属性省略，编译器将使用默认设置

    "noUncheckedIndexedAccess": false,  // 从索引操作（如 arr[0]）中获取的值不会是 undefined。如果开启（true），TypeScript 会认为每次通过索引访问都可能返回 undefined，增加安全性但也增加代码复杂度
    "noImplicitOverride": true,  //要求子类中使用 override关键字来重写父类方法，确保重写的方法确实存在于父类中，避免拼写错误或误解
    "exactOptionalPropertyTypes": false // 如果开启（true），那么一个被定义为可选属性（如 { prop?: string }）将不允许被显式设置为 undefined（只能是不定义该属性或赋予一个字符串值）。关闭此选项则更灵活
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist", "**/*.test.ts"]
}
