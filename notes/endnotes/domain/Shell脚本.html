Shell脚本学习笔记

禹穆穆2024/12/15...大约 8 分钟

Shell历史发展

起源和发展

Shell 的起源与早期发展 Shell 的本质是用户与操作系统内核（Kernel）之间的桥梁。它接收用户输入的命令，将其转换为系统调用，交由内核执行，最后把结果返回给用户。早期的计算机交互方式深受电传打字机（Teletypewriter, TTY）​​ 的影响。在图形显示器普及之前，电传打字机是主要的计算机终端，用户通过键盘输入命令，计算机会将输出内容打印在纸上。这种工作方式塑造了早期 Shell 的许多特性，例如行缓冲输出、简洁的命令语法​（为了节省纸张和时间）以及一些控制字符​（如 Ctrl+C中断）的定义，至今，Linux/Unix 系统中仍用 ​TTY​ 来指代终端设备，在 Unix 系统诞生之初，Ken Thompson 于 1971 年开发了第一个 Unix shell，即 ​Thompson shell​，它主要作为一个交互式命令解释器，允许用户输入命令并查看结果，虽然它引入了重定向<, >, >>和管道|等核心概念，但其脚本能力非常有限，几乎只能执行简单的命令序列，缺乏变量、控制流等编程元素，这严重限制了自动化任务的实现

重要 Shell 的演变历程

  1. Bourne shell (sh) - 1977​：由 Stephen Bourne 在贝尔实验室开发。它被设计用来解决 Thompson shell 的脚本编写局限性。Bourne shell 引入了变量、控制流（如 if-then-else 条件判断、for/while 循环）、函数（尽管最初功能有限）以及命令替换等特性，使其成为第一个真正具备强大脚本编程能力的 Unix shell​。它迅速成为 Unix 系统的标准 shell，许多系统管理脚本都基于它编写。

  2. C shell (csh) - 1978​：由 Bill Joy 在伯克利分校开发。其语法设计借鉴了 C 语言，对 C 程序员更友好。它的主要贡献在于极大地增强了交互式体验，引入了命令历史记录、别名（alias）、作业控制（如后台运行、前后台切换）​​ 等功能。但其脚本语法与 Bourne shell 不兼容，且早期版本在脚本执行的稳定性和效率上存在一些问题，因此更多被用于交互，而非脚本编写。

  3. Korn shell (ksh) - 1983​：由 David Korn 在 AT&T 贝尔实验室开发。ksh 旨在融合 Bourne shell 的脚本优势和 C shell 的交互优势​。它完全向上兼容 Bourne shell​，同时加入了命令历史、行内编辑、更强大的编程功能（如关联数组）等特性。ksh 最初是商业软件，这在一定程度上影响了其普及。

  4. Bash (Bourne-Again Shell) - 1989​：由 Brian Fox 为自由软件基金会（FSF）的 GNU 项目开发。Bash 的设计目标是完全兼容 Bourne shell (sh)​，同时融入 C shell 和 Korn shell 的诸多实用特性​。它提供了命令历史、命令行编辑、别名、作业控制、数组、花括号扩展、条件测试增强等功能。由于其开源免费、功能强大且兼容性好，Bash 逐渐成为大多数 Linux 发行版和 macOS（2019 年前）的默认 shell​，至今仍是世界上最流行的 Shell 之一

Shell 的类型与差异

根据语法和特性，Shell 大致可分为两大系列：

  • Bourne shell 系列​：包括 sh、bash、ksh、zsh。它们的基本语法和脚本结构相似，具有较好的兼容性
  • ​C shell 系列​：包括 csh、tcsh。其语法类似 C 语言，但与其他 shell 的脚本兼容性较差

核心语法

脚本基础结构

  • 一个 Shell 脚本通常以 ​Shebang​ (#!) 开头，用于指定解释器：#!/bin/bash

变量

  • 定义和使用：变量赋值时等号两边不能有空格，使用 $或 ${}引用变量
name="Alice"  # 定义变量
echo "Hello, $name"  # 使用变量：Hello, Alice
echo "Hello, ${name}!"  # 使用变量：Hello, Alice!
  • 环境变量：如 $HOME、$PATH、$PWD等，可用于脚本中
  • 位置参数：脚本或函数接收的参数，$0表示脚本名，$1、$2... 表示第1、2...个参数，$@表示所有参数（数组），$#表示参数个数
  • ​特殊变量​：如$?获取上一个命令的退出状态（0 表示成功），$$获取当前脚本的进程 ID

字符串操作

  • Shell 脚本中的变量通常被视为字符串，并支持一些操作
str="Hello, World!"
echo ${#str}      # 输出字符串长度：13
echo ${str:7:5}  # 从第7个字符开始截取5个字符：World
echo ${str/World/Shell}  # 替换第一次出现的World为Shell：Hello, Shell!

数学运算

  • $(( ))进行整数运算
result=$((5 + 3))  # 结果为8
  • expr命令也可进行整数运算
result=$(expr 5 + 3)  # 结果为8
  • 对于更复杂的运算（如浮点数），常借助 bc等外部工具

条件判断

  • 使用 if、elif、else和 test命令（或 [ ]）进行条件判断
  • ​文件测试​
if [ -f "/path/to/file" ]; then  # 判断是否是文件
    echo "File exists."
fi
  • 字符串比较
if [ "$str1" == "$str2" ]; then  # 判断字符串相等
    echo "Strings are equal."
fi
if [ -z "$str" ]; then  # 判断字符串长度是否为0（空）
    echo "String is empty."
fi
if [ -n "$str" ]; then  # 判断字符串长度是否非0（非空）
    echo "String is not empty."
fi
  • 数值比较​：使用 -eq（等于）、-ne（不等于）、-gt（大于）、-ge（大于等于）、-lt（小于）、-le（小于等于）
if [ $num1 -gt $num2 ]; then
    echo "$num1 is greater than $num2."
fi
  • 逻辑操作​：&&（与）、||（或）用于连接多个条件
if [ -f "$file" ] && [ -r "$file" ]; then  # 文件存在且可读
    echo "File is readable."
fi
  • case语句
read -p "Enter your choice: " choice
case $choice in
    1) echo "You chose 1." ;;
    2) echo "You chose 2." ;;
    *) echo "Invalid choice." ;;
esac

循环结构

  • for 循环​：遍历列表或范围
for i in 1 2 3 4 5; do  # 或 for i in {1..5}; do
    echo "Number: $i"
done
for file in *.txt; do  # 遍历当前目录所有txt文件
    echo "Processing $file"
done
  • while循环：当条件为真时执行循环
count=1
while [ $count -le 5 ]; do
    echo "Count: $count"
    ((count++))  # 或 count=$((count+1))
done
  • until 循环​：直到条件为真时停止循环
count=1
until [ $count -gt 5 ]; do
    echo "Count: $count"
    ((count++))
done

函数

  • 定义和调用
function greet() {  # function关键字可省略
    echo "Hello, $1!"  # $1表示函数的第一个参数
}
greet "Alice"  # 调用函数，输出：Hello, Alice!
  • ​返回值​：函数默认返回最后一条命令的退出状态，也可用 return返回特定数值（0-255）通常使用 echo输出结果，并通过命令替换 $(greet "Alice")捕获

输入输出重定向与管道

  • 重定向
    • >：将标准输出重定向到文件
    • >>：将标准输出重定向到文件（追加）
    • 2>：将标准错误重定向到文件
    • &>：将标准输出和标准错误都重定向到文件
    • <：从文件读取内容作为标准输入
  • 管道|​：将一个命令的输出作为另一个命令的输入
cat /var/log/syslog | grep "ERROR" | wc -l  # 统计syslog中ERROR的行数

数组操作

fruits=("apple" "banana" "orange")  # 定义数组
echo ${fruits[0]}  # 访问第一个元素：apple
echo ${fruits[@]}  # 访问所有元素：apple banana orange
echo ${#fruits[@]}  # 获取数组长度：3
for fruit in "${fruits[@]}"; do  # 遍历数组
    echo "$fruit"
done

脚本调试与错误处理

  • 调试模式​：
    • 脚本开头加 set -x或在运行脚本时使用 bash -x script.sh，会显示每个命令及其参数
    • bash -n script.sh只检查语法错误而不执行
  • 错误处理
    • set -e：脚本中任何命令返回非零状态（失败）时立即退出
    • set -u：遇到未定义的变量时报错并退出
    • trap命令用于捕获信号并执行特定命令，可用于脚本退出前的清理工作
cleanup() {
    echo "Cleaning up..."
    # 删除临时文件等操作
}
trap cleanup EXIT  # 在脚本退出时执行cleanup函数

复杂案例

  • 自动化备份脚本
#!/bin/bash
# 定义源目录和备份目录
src_dir="/home/user/documents"
backup_dir="/backup"
# 使用日期作为备份文件名的一部分
backup_file="backup_$(date +%Y%m%d).tar.gz"

# 检查源目录是否存在
if [ ! -d "$src_dir" ]; then
    echo "Error: Source directory $src_dir does not exist."
    exit 1
fi

# 创建备份压缩包
tar -czf "$backup_dir/$backup_file" "$src_dir"

# 检查tar命令是否成功
if [ $? -eq 0 ]; then
    echo "Backup completed: $backup_file in $backup_dir"
else
    echo "Backup failed."
    exit 1
fi
  • 系统监控脚本
#!/bin/bash
# 定义日志文件路径和阈值
LOG_FILE="/var/log/system_monitor.log"
CPU_THRESHOLD=80
MEM_THRESHOLD=90

# 获取CPU使用率（使用top命令）
cpu_usage=$(top -bn1 | grep "Cpu(s)" | sed "s/.*, *\([0-9.]*\)%* id.*/\1/" | awk '{print 100 - $1}')
# 获取内存使用率（使用free命令）
mem_usage=$(free | grep Mem | awk '{print $3/$2 * 100.0}')
# 获取当前时间
current_time=$(date '+%Y-%m-%d %H:%M:%S')

# 记录到日志文件
echo "[$current_time] CPU Usage: $cpu_usage% | Memory Usage: $mem_usage%" >> $LOG_FILE

# 检查是否超过阈值（使用bc进行浮点数比较）
if (( $(echo "$cpu_usage > $CPU_THRESHOLD" | bc -l) )); then
    echo "[$current_time] WARNING: CPU usage exceeded $CPU_THRESHOLD% (Current: $cpu_usage%)" >> $LOG_FILE
fi

if (( $(echo "$mem_usage > $MEM_THRESHOLD" | bc -l) )); then
    echo "[$current_time] WARNING: Memory usage exceeded $MEM_THRESHOLD% (Current: $mem_usage%)" >> $LOG_FILE
fi
  • 自动化部署脚本
#!/bin/bash
set -e  # 遇到错误立即退出脚本，避免继续执行可能造成更大问题[5](@ref)

# 定义变量
APP_DIR="/var/www/myapp"
GIT_REPO="https://github.com/username/myapp.git"
BRANCH="main"

echo "Starting deployment process..."

# 检查并进入应用目录
if [ ! -d "$APP_DIR" ]; then
    echo "Cloning repository..."
    sudo git clone "$GIT_REPO" "$APP_DIR"
fi

cd "$APP_DIR"

# 获取最新代码
echo "Pulling latest code from $BRANCH branch..."
sudo git fetch origin
sudo git checkout "$BRANCH"
sudo git reset --hard origin/"$BRANCH"  # 强制重置到远程分支最新状态

# 安装依赖并构建（假设是Node.js项目）
echo "Installing dependencies and building..."
sudo npm install  # 假设使用npm
sudo npm run build

# 重启服务（例如Nginx）
echo "Restarting Nginx..."
sudo systemctl restart nginx

echo "Deployment completed successfully!"
