Linux快速入门

什么是Linux

  • Linux是一个开源的类Unix操作系统，最初由Linus Torvalds于1991年发布。它是基于Unix设计的，广泛用于服务器、桌面和嵌入式系统。

相关分类

Linux的版本（或发行版）种类繁多，主要可以按照不同的标准进行分类。以下是一些常见的Linux发行版分类：

1. 根据用途分类

  • 桌面版：适合个人用户和日常使用，提供图形用户界面（GUI）。

    • 例：Ubuntu、Fedora、Linux Mint、Zorin OS等。

  • 服务器版：优化用于服务器环境，通常不带图形界面，重视安全性和稳定性。

    • 例：CentOS、Debian、Ubuntu Server、Red Hat Enterprise Linux (RHEL)等。

  • 嵌入式版：专为嵌入式系统设计，通常体积小巧。

    • 例：Yocto、OpenWrt、Raspberry Pi OS等。

2. 根据包管理系统分类

  • Debian系：使用.deb包格式，主要通过apt管理软件包。

    • 例：Debian、Ubuntu、Linux Mint等。

  • Red Hat系：使用.rpm包格式，主要通过yum或dnf管理软件包。

    • 例：Red Hat Enterprise Linux (RHEL)、CentOS、Fedora等。

  • Arch系：采用滚动更新模式，用户可以通过pacman管理软件包。

    • 例：Arch Linux、Manjaro等。

3. 根据开发模型分类

  • 社区版：由社区开发和维护，通常更新较快，适合喜欢尝试新技术的用户。

    • 例：Fedora、openSUSE等。

  • 企业版：由公司开发和维护，提供长期支持和商业服务，适合企业使用。

    • 例：Red Hat Enterprise Linux (RHEL)、SUSE Linux Enterprise Server (SLES)等。

4. 根据用户友好程度分类

  • 用户友好型：针对新手用户，提供简化的安装和配置过程。

    • 例：Ubuntu、Linux Mint、Zorin OS等。

  • 高级用户型：适合经验丰富的用户，提供更多的定制选项和控制。

    • 例：Arch Linux、Gentoo等。

5. 根据特定用途分类

  • 安全性导向：专注于安全性和隐私保护。

    • 例：Tails、Qubes OS等。

  • 教育用途：专为教育和学习设计。

    • 例：Edubuntu、Sugar on a Stick等。

这些分类只是对Linux发行版的一种简单概述，实际上还有许多其他的发行版和变种，可以根据自己的需求选择合适的版本

在win11中安装Linux系统

  • 安装方式：1.用win11的WSL2安装（win11非家庭版） 2.下载虚拟机vmware安装。这里采用方式1安装Linux，方式2自行搜索安装
  • 通过win11安装ubuntu
    • 开启适用于linux的windows子系统：控制面板>程序>启用或关闭 Windows功能，勾选a.适用于 Linux的 Windows子系统 b.虚拟化平台
    • 按照系统提示是否要求重启系统
    • 设置默认的wsl2版本：wsl --set-default-version 2
    • 打开终端，输入指令wsl --list --online，查看有哪些可以安装的系统
    • 安装Ubuntu-20.04：wsl --install -d Ubuntu-20.04
    • 现有WSL升级：wsl --update
    • 关闭wsl：wsl --shutdown
    • 在终端的下拉标签中选择Ubuntu-20.04，如果没有可以重新打开一个终端查看，设置用户名和密码后使用ubuntu系统

初始目录

  • 根目录
    • bin: 存放普通用户执行的指令
    • dev: 设备目录，用于存放设备相关文件
    • boot: 开机引导目录，包括Linux内核文件与开机所需要的文件
    • home：存放用户主数据
    • usr: 应用程序放置目录
    • lib: 开机常用的动态链接库，bin及sbin指令也会调用对应的lib库
    • tmp: 临时文件目录
    • etc: 系统配置文件目录，大部分配置属性均放在这里

相关常用Linux命令（ubuntu）

  • 查看linux命令：https://github.com/jaywcjlove/linux-command
  • 查看当前linux版本：cat /etc/issue 或者 cat /etc/redhat-release 或者 uname -a
  • 目录及文件
    • cd / pwd
    • ls / ls -a
    • ls -l
    • cat / cat -n
    • mkdir / rm -rf
    • mv / cp
    • touch
    • tree：查看目录树，但系统本身并没有安装tree命令，使用sudo apt install tree指令安装
    • grep + <关键字>: 在当前目录下查找包含关键字的文件
    • vim: 三种模式
      • 命令模式：只能查看文件内容，不能编辑，按i、o、a键可以进入输入模式，按:键可以进入底线命令模式
      • 输入模式：可以编辑文件内容，按ESC键可以进入命令模式
        • 删除当前行：dd
        • 行号显示和取消：:set nu、:set nonu
        • 复制：yy、p
        • 查找：/、n、N
      • 底线命令模式：可以保存退出，q、wq、wq!
      • vim插件安装
    • 管道符：将一个命令的执行结果作为另一个命令的输入来执行，格式为命令1 | 命令2
  • 进程管理
    • ps aux：查看系统中所有进程，使用BSD操作系统格式
    • ps -le: 查看系统中所有进程，使用Linux操作系统格式
    • 使用ps输出内容后，输入内容格式含义
      • USER：进程所有者
      • PID：进程ID
      • %CPU：进程占用的CPU使用率
      • %MEM：进程占用的物理内存百分比
      • VSZ：进程使用的虚拟内存大小（KB）
      • RSS：进程使用的物理内存大小（KB）
      • TTY：终端的次要设备号（通常在终端使用）
      • STAT：进程状态。D表示不可中断的睡眠状态，R表示正在运行，S表示处于休眠状态，T表示停止或被追踪，Z表示僵尸进程
      • START：进程启动时间
      • TIME：进程运行时间
      • COMMAND：进程启动的命令名称
    • pstree：显示系统中进程的树形结构
    • top：动态显示系统中各个进程的资源占用状况，按q退出
    • kill：终止指定的进程，格式为kill -9 <PID>
  • 用户及用户组管理
    • 安装时的账号切换到root账号
      • ubuntu默认root用户无固定密码，每次开机都有一个新密码，通过sudo passwd root指令设置root用户密码
      • 切换到root用户：su root
      • 退出root用户：exit
    • 添加用户：useradd 选项 用户名
      • 选项：-c 备注信息，-d 指定根目录，-g 所属组，-G 所属附加组，-m 自动创建根目录，-s 登录shell
    • 删除用户：userdel 选项 用户名
      • 选项：-r 同时删除用户的主目录
    • 更新用户：usermod 选项 用户名
      • 选项：与添加用户相同
    • 查看所有用户：cat /etc/passwd
    • 用户口令：passwd 选项 用户名
      • -l锁定口令（禁用账号），-u解锁口令，-d删除口令，-f 强迫用户下次登录时修改口令
      • 如果是root用户，使用passwd命令修改root用户口令
  • 权限管理
    • 文件和目录的权限分为三种：读（r）、写（w）、执行（x），例如，权限-rwxr-xr--表示：
      • -：表示这是一个文件（如果是目录则为d）。
      • rwx：文件所有者有读、写和执行权限。
      • r-x：同组用户有读和执行权限，但没有写权限。
      • r--：其他用户只有读权限。
    • chmod：修改文件权限
      • chmod 选项 权限值 文件名
      • 选项：-R 递归修改子目录及文件权限
      • 权限值：u、g、o、a、+、-、=
      • u：文件所有者
      • g：文件所属组
      • o：其他用户
      • a：所有用户
  • 磁盘管理
    • df：查看磁盘空间使用情况
      • df - h：以人类可读的方式显示磁盘空间使用情况
      • df -T：显示文件系统类型

第一部分：Linux的规则与安装

0、计算机概论

  • 什么是计算机：接受用户输入的命令与数据，经由中央处理器的算术与逻辑单元运算处理后，产生或存储有用的信息。
  • 电脑硬件的五大单元（以个人电脑为例）：
    • 输入单元：键盘、鼠标、扫描仪、光电输入设备
    • 主机部分：系统单元，（主板）被主机机箱保护，CUP内部的控制单元和算术逻辑单元，内存
    • 输出单元：屏幕、打印机、音响等
  • 五大单元运行流程：
    • CPU读取数据都是从内存来的，内存中的数据则是从输入单元所传输进来的，而CPU处理完毕的数据也必须先写回内存，最后数据才从内存传输到输出单元
    • 算术逻辑单元主要负责程序运算与逻辑判断，控制单元则主要协调各个周报组件与各个单元的工作
根据电脑型号配置计算速度瓶颈
  • 基本信息：
    • cpu型号：Intel Core i7 13620H， 10核
    • 内存：DDR5-5600(2800MHz)，2个插槽各8G共16G内存大小
    • 显卡：型号RTX4050，显存7001MHz大小6G位宽96位，PCLe 4.0 x8
    • 磁盘：大小1024G，传输模式PCIe 4.0 x4
    • 网卡：千兆网络，即125Mb/s的传输速度
  • 计算电脑整体速度瓶颈
    • 内存：计算公式带宽=频率 * 位宽，2800 * 128 / 8 /1000 = 44.8GB/s
    • 显卡：PCLe 4.0 x8对应传输速率为16GB/s
    • 磁盘：PCIe 4.0 x4对应传输速率为8GB/s
    • 网卡：千兆网络，最高125Mb/s的传输速度
    • 总结：速度瓶颈在网络方面

第二部分：Linux文件、目录与磁盘格式

1、文件权限和目录配置

  • 用户和用户组：用户类似你自己的房间，有些东西别入不能动；用户组类似你的家，有公共区域
  • root用户：超级用户，权限最高
  • 默认情况下，账号相关文件存放
    • 用户信息：/etc/passwd
    • 个人密码：/etc/shadow
    • 用户组信息：/etc/group
  • 文件权限：

    • 权限：r 表示可读（4），w表示可写（2），x表示可执行（1），以-rw-r--r-- 1 root root为例，表示这个文件属于root用户，root用户有读写权限，root用户组只有读的权限，其他用户也只有读的权限，这组权限的数字权限用644表示
    • 修改文件属性和权限：
# 都可以加入参数-R，表示递归修改，子目录下所有文件都更新为这个用户/用户组
chgrp groupname filename/dirname    # 修改用户组
chown user    filename #修改文件归属用户
chmod xyz  filename/dirname  # 修改文件的权限，xyz表示对应的数字权限，最高为777
  • 文件权限和扩展名：
    • 常规文件：
      • 纯文本文件（ASCLL），执行cat命令可以直接查看
      • 二进制文件（binary），可执行文件就属于这种格式，如cat
      • 数据文件（data），读取某些特定格式文件，比如登录的数据记录在/var/log/wtmp文件中，这个数据文件使用cat读取会乱码，但通过last命令可以正常读取
    • 目录（directory）
    • 链接文件(link)，类似win11中的快捷方式
    • 设备与设备文件（device）：通常都放在/dev目录下，分如下两种
      • 区块设备文件：存储数据时可以随机在不同区块读写，查看/dev/sda，第一个属性为b
      • 字符设备文件：串行端口的接口设备，如键盘、鼠标，第一个属性为c
    • 数据接口文件（sockets）：通常用于网络上的数据交换，一般在/run或/tmp目录中，第一个属性为c
    • 数据输送文件（FIFO，pipe）：可以解决多个程序同时读写一个文件造成的错误问题，先进先出（管道），第一个属性为p
    • 文件扩展名：一个文件是否被执行，和文件扩展名没有任何关系，而是取决于是否有x可执行权限。注：可执行权限和可执行的程序代码是两回事，比如text.txt文件被赋予x权限，但不具备可执行代码，而cat命令取消x权限将无法被执行
      • 因为可执行权限但不一定执行成功，所有通常还是会以文件扩展名区分种类，比如.sh、.tar、.zip等文件扩展名
