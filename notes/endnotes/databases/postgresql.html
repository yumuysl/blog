为什么使用PostgraSQL?

PostgreSQL（通常简称为Postgres）是一种开源的对象关系数据库管理系统，与其他数据库系统相比，它有其独特的优缺点。以下是PostgreSQL的一些主要优缺点：

优点：

  1. 开源与社区支持：

    • PostgreSQL是开源的，用户可以自由使用、修改和分发。拥有活跃的社区支持，提供丰富的文档和资源。

  2. 强大的功能：

    • 支持复杂的查询、事务、完整性约束、视图、存储过程等高级功能。
    • 提供对JSON数据类型的良好支持，适合处理非结构化数据。

  3. 可扩展性：

    • 可以通过插件和扩展来增加功能，比如PostGIS用于地理空间数据。
    • 支持自定义数据类型和函数。

  4. ACID兼容性：

    • 确保事务的原子性、一致性、隔离性和持久性，适合对数据一致性有高要求的应用。

  5. 多种索引类型：

    • 支持多种索引类型（如B-tree、Hash、GiST、GIN等），可以优化查询性能。

  6. 跨平台支持：

    • 可以在多种操作系统上运行，包括Linux、Windows和macOS。

缺点：

  1. 性能问题：

    • 在某些特定场景下，PostgreSQL的性能可能不如某些专有数据库（如Oracle或SQL Server），尤其是在高并发写入操作时。

  2. 学习曲线：

    • 对于新手来说，PostgreSQL的功能和配置选项可能会让人感到复杂，学习曲线较陡峭。

  3. 资源消耗：

    • 相比于一些轻量级数据库（如SQLite），PostgreSQL可能需要更多的系统资源（内存和CPU）。

  4. 默认配置：

    • 在某些情况下，PostgreSQL的默认配置可能不适合生产环境，需要进行调优以达到最佳性能。

  5. 社区支持的限制：

    • 尽管有活跃的社区，但在某些情况下，缺乏商业支持可能会让企业在遇到关键问题时感到不安。

总的来说，PostgreSQL是一款功能强大且灵活的数据库系统，适合需要复杂查询和高数据一致性的应用。然而，在选择数据库时，最好根据具体需求和使用场景进行综合考虑。

PostgraSQL在Node.js中的使用

  • postgreSql的的SQL语句是最靠近标准的SQL协议的

  • 连接数据库并设置权限，需要用到pg模块

    const { Pool } = require('pg');

const pool = new Pool({
  user: 'your_username',
  host: 'localhost',
  database: 'your_database',
  password: 'your_password',
  port: 5432,
});

pool.query('SELECT NOW()', (err, res) => {
  if (err) {
    console.error('Error connecting to database', err);
  } else {
    console.log('Connected to database');
  }
});

  • 创建表

    pool.query('CREATE TABLE users (id SERIAL PRIMARY KEY, name VARCHAR(50), email VARCHAR(50))', (err, res) => {
if (err) {
  console.error('Error creating table', err);
} else {
  console.log('Table users created');
}
});

  • 删除表

    pool.query('DROP TABLE IF EXISTS users', (err, res) => {
if (err) {
  console.error('Error dropping table', err);
} else {
  console.log('Table users dropped');
}
});

  • 设置表

    pool.query('ALTER TABLE users ADD COLUMN age INT', (err, res) => {
if (err) {
  console.error('Error adding column', err);
} else {
  console.log('Column age added to table users');
}
});

  • 增删查改

    // 插入数据
pool.query('INSERT INTO users (name, email, age) VALUES ($1, $2, $3)', ['Alice', 'alice@example.com', 30]);

// 查询数据
pool.query('SELECT * FROM users', (err, res) => {
  if (err) {
    console.error('Error querying data', err);
  } else {
    console.log(res.rows);
  }
});

// 更新数据
pool.query('UPDATE users SET age = $1 WHERE name = $2', [25, 'Alice']);

// 删除数据
pool.query('DELETE FROM users WHERE name = $1', ['Alice']);

表的一些基本设置

数据表基本字段

  • id：设置为主键
  • uuid：需要在该数据库中安装uuid-ossp扩展，通过该扩展的uuid_generate_v4()方法可以设置uuid，建议设置为初始默认
  • created_time：创建时间，字段类型为timestemp without time zone，默认值CURRENT_TIMESTAMP
  • motify_time：修改时间，默认值同上，需要自动更新该字段，可以新建触发器函数，设置中的代码如下，然后给该表设置触发器，选择该触发器函数
BEGIN
	NEW.motify_time = CURRENT_TIMESTAMP;
	RETURN NEW;
END
  • is_active：是否激活，用于伪删除，默认值true

相关数据库操作封装工具（AI总结）

在 Node.js 中，有许多工具库封装了对数据库的增删查改（CRUD）操作以及对事务和锁的操作。以下是一些常用的库和 ORM（对象关系映射）工具：

1. Sequelize

  • 描述：Sequelize 是一个基于 Promise 的 Node.js ORM，支持多种数据库（如 PostgreSQL、MySQL、MariaDB、SQLite 和 Microsoft SQL Server）。
  • 功能：
    • 提供 CRUD 操作的简单接口。
    • 支持事务管理。
    • 支持模型定义、关联、验证和钩子等功能。
const { Sequelize, DataTypes } = require('sequelize');
const sequelize = new Sequelize('database', 'username', 'password', {
    dialect: 'postgres',
});

// 定义模型
const User = sequelize.define('User', {
    username: {
        type: DataTypes.STRING,
        allowNull: false,
    },
});

// 创建事务
await sequelize.transaction(async (t) => {
    const user = await User.create({ username: 'example' }, { transaction: t });
});

2. TypeORM

  • 描述：TypeORM 是一个支持 TypeScript 和 JavaScript 的 ORM，支持多种数据库。
  • 功能：
    • 提供强大的 CRUD 接口。
    • 支持复杂的查询、事务、关系和数据迁移。
    • 使用装饰器定义模型。
import { Entity, PrimaryGeneratedColumn, Column, createConnection, getConnection } from 'typeorm';

@Entity()
class User {
    @PrimaryGeneratedColumn()
    id: number;

    @Column()
    username: string;
}

// 创建事务
await getConnection().transaction(async (entityManager) => {
    const user = new User();
    user.username = 'example';
    await entityManager.save(user);
});

3. Mongoose

  • 描述：Mongoose 是一个 MongoDB 的对象建模工具，提供了对 MongoDB 的 CRUD 操作。
  • 功能：
    • 提供 Schema 定义和验证功能。
    • 支持事务（MongoDB 4.0 及以上版本）。
const mongoose = require('mongoose');

const userSchema = new mongoose.Schema({
    username: String,
});

const User = mongoose.model('User', userSchema);

// 创建事务
const session = await mongoose.startSession();
session.startTransaction();
try {
    const user = new User({ username: 'example' });
    await user.save({ session });
    await session.commitTransaction();
} catch (error) {
    await session.abortTransaction();
} finally {
    session.endSession();
}

4. prisma

  • 描述：prisma是一个ORM工具，支持多种数据库，包括PostgreSQL。

  • 功能：

    • 提供了ORM的功能，支持多种数据库，包括PostgreSQL。
    • 提供了数据库的迁移功能，方便数据库的版本控制。
    • 提供了数据库的种子数据功能，方便数据库的初始化。
    • 提供了数据库的测试数据功能，方便数据库的测试。

  • 使用方法：

    • 官网：https://prisma.org.cn/docs/orm/prisma-schema/data-model/unsupported-database-features
    • 安装prisma：pnpm install prisma --save-dev ，pnpm install @prisma/client
    • 编辑.env文件，更新URL
    • 初始化prisma：pnpm prisma init
    • 配置prisma：prisma/schema.prisma
      • 该文件需要与数据库字段格式一致
      • 方式：
        • 内省：pnpm prisma db pull，注意内省后需要pnpm prisma generate更新客户端
        • 迁移：手动配置prisma/schema.prisma文件，然后迁移prisma migrate dev --name init，注意该命令会自动pnpm prisma generate更新客户端

  • 相关数据库操作及配置查看官网

数据表事件触发和定时任务(AI总结)

在Node.js中，可以使用PostgreSQL的触发器（Triggers）和计划任务（Scheduled Events）来处理触发事件和定期执行事件。以下是如何在PostgreSQL中设置这两种事件的详细步骤。

1. 设置触发器（Triggers）

触发器是在特定事件（如插入、更新或删除）发生时自动执行的函数。以下是设置触发器的步骤：

步骤 1：创建触发器函数

首先，创建一个触发器函数，它将在特定事件发生时执行。例如，以下是一个在插入新记录时记录日志的触发器函数：

CREATE OR REPLACE FUNCTION log_insert()
RETURNS TRIGGER AS $$
BEGIN
    INSERT INTO logs (message) VALUES ('Inserted a new record with id: ' || NEW.id);
    RETURN NEW;
END;
$ LANGUAGE plpgsql;

步骤 2：创建触发器

然后，创建一个触发器，将其与表关联。例如，以下是一个在插入 example 表时触发的触发器：

CREATE TRIGGER after_insert_example
AFTER INSERT ON example
FOR EACH ROW
EXECUTE FUNCTION log_insert();

2. 设置定期执行事件

PostgreSQL本身没有内置的定时任务调度功能，但可以使用一些扩展，比如 pg_cron ，或者使用外部调度工具（如 cron 或 node-cron ）来实现定期执行事件。

方法 1：使用 pg_cron （PostgreSQL扩展）

pg_cron 是一个 PostgreSQL 扩展，允许你在数据库中安排定期任务。

步骤 1：安装 pg_cron

在PostgreSQL中安装 pg_cron 扩展： CREATE EXTENSION pg_cron;

步骤 2：创建定期任务

使用 SELECT cron.schedule 来创建定期任务。例如，每小时执行一次某个函数： SELECT cron.schedule('0 * * * *', CALLyourfunction()CALL your_function()CALLyourf​unction());

方法 2：使用 Node.js 定时任务

如果你更倾向于在应用层面控制定时任务，可以使用Node.js库，如 node-cron 。

步骤 1：安装 node-cron
npm install node-cron
步骤 2：设置定时任务

在你的Node.js应用中，使用 node-cron 来设置定时任务。例如，每小时执行一次某个数据库操作：

const cron = require('node-cron');
const { Client } = require('pg');

const client = new Client({
    user: 'your_user',
    host: 'localhost',
    database: 'your_database',
    password: 'your_password',
    port: 5432,
});

client.connect();

cron.schedule('0 * * * *', async () => {
    // 执行数据库操作
    await client.query('CALL your_function()');
    console.log('Function executed every hour');
});
